# 脚本执行文档

本文档仅记录当前仍保留在主流程中的脚本。

## 1. 数据脚本

### scripts/data/prepare_flir.py

用途：整理 FLIR 数据集，生成训练/验证所需目录结构。

### scripts/data/images_to_video.py

用途：将同一热视频序列的图像帧合成为 mp4 视频，供跟踪评估使用。

## 2. 训练脚本

### scripts/train/train_yolov5.py

用途：训练单个检测模型，是对 yolov5/train.py 的精简封装。

常用示例：

```bash
python scripts/train/train_yolov5.py --config configs/train_config.yaml
```

说明：

- 该脚本只负责读取配置、校验路径并启动训练。
- 不再自动克隆 YOLOv5，也不再自动下载预训练权重。
- 如需修改训练参数，优先改 configs/train_config.yaml，必要时再用命令行覆盖。

### scripts/train/train_ablation.py

用途：批量执行消融实验；通过 profile 配置管理 controlled 与 optimal 两种训练口径。

常用示例：

```bash
python scripts/train/train_ablation.py --profile controlled
```

```bash
python scripts/train/train_ablation.py --profile optimal
```

说明：

- train_ablation.py 不读取 configs/train_config.yaml。
- controlled 模式默认读取 configs/ablation/train_profile_controlled.yaml。
- optimal 模式默认读取 configs/ablation/train_profile_optimal.yaml。
- 如需临时改 epochs、batch、img，只作为本次命令覆盖，不应回写成单模型训练配置。

## 3. 评估脚本

### scripts/evaluate/eval_detection.py

用途：

- 单模型检测指标评估
- speed 模式测速
- 批量消融评估

配置边界：

- 默认读取 configs/eval_detection.yaml
- 命令行参数只做本次运行覆盖
- 默认在 outputs/detection/批次目录 下输出 summary.json
- batch-eval 默认同目录输出 summary.csv
- metric 评估不再单独生成 outputs/val_detection 这类额外根目录

常用示例：

```bash
python scripts/evaluate/eval_detection.py --config configs/eval_detection.yaml --weights outputs/ablation_study/ablation_exp07_eiou/weights/best.pt
```

```bash
python scripts/evaluate/eval_detection.py --config configs/eval_detection.yaml --batch-eval --save-csv --sort-by map5095
```

### scripts/evaluate/eval_tracking.py

用途：对指定检测模型和跟踪器跑视频批量测试。

配置边界：

- 默认读取 configs/tracking_config.yaml
- 配置文件负责检测器输入、输出开关和各 tracker 默认参数
- 命令行参数只做本次运行覆盖

常用示例：

```bash
python scripts/evaluate/eval_tracking.py --config configs/tracking_config.yaml --weights outputs/ablation_study/ablation_exp07_eiou/weights/best.pt --tracker deepsort --output outputs/tracking/current --half --no-save-vid --no-save-txt --no-overlay
```

### scripts/evaluate/plot_eval_summary.py

用途：为检测消融结果生成汇总图。

配置边界：

- 默认读取 configs/plot_eval_summary.yaml
- 命令行参数只做本次运行覆盖
- 默认从某次 detection 批次目录读取 summary.csv
- 默认在同目录写入 journal_barline.png、journal_improvement.png、journal_metric_change.png

常用示例：

```bash
python scripts/evaluate/plot_eval_summary.py --config configs/plot_eval_summary.yaml --input-dir outputs/detection/detection_eval_batch_xxx
```

### scripts/evaluate/plot_tracking_algorithm_comparison.py

用途：读取多个跟踪器的 summary_metrics.json，生成对比表和图。

配置边界：

- 默认读取 configs/plot_tracking_comparison.yaml
- 命令行参数只做本次运行覆盖

常用示例：

```bash
python scripts/evaluate/plot_tracking_algorithm_comparison.py --config configs/plot_tracking_comparison.yaml
```
