# 完整实施指南

## 当前范围

本指南只覆盖当前仍在仓库内维护的实验链路：

- 数据准备
- 检测训练
- 检测消融评估
- 多目标跟踪评估
- 跟踪结果汇总

部署与嵌入式相关内容已经从当前仓库移除。

## 一、数据准备

1. 整理 FLIR 数据集到 data/raw/flir
2. 运行 prepare_flir.py 生成 data/processed/flir
3. 如需跟踪实验，整理热视频到 data/videos/thermal_test

## 二、检测训练

### 基准训练

```bash
python scripts/train/train_yolov5.py --config configs/train_config.yaml
```

训练前确认：

1. yolov5/train.py 存在
2. configs/train_config.yaml 中的数据路径正确
3. 默认预训练权重 yolov5/yolov5s.pt 存在

### 消融训练

```bash
python scripts/train/train_ablation.py --profile controlled
```

如需以“各实验上限性能”为目标：

```bash
python scripts/train/train_ablation.py --profile optimal
```

建议遵循控制变量原则：

1. 先单变量实验
2. 再组合实验
3. 所有对比实验保持同一训练口径

配置边界：

1. 单模型训练只改 configs/train_config.yaml
2. 消融实验只改 configs/ablation/train_profile_controlled.yaml 或 configs/ablation/train_profile_optimal.yaml
3. 不要把单模型训练参数混到消融 profile 文件之外

## 三、检测评估

### 单模型评估

```bash
python scripts/evaluate/eval_detection.py --mode metric --weights outputs/ablation_study/ablation_exp07_eiou/weights/best.pt --data data/processed/flir/dataset.yaml --batch-size 16 --img-size 640 --device 0 --workers 8 --save-json
```

### 批量评估

```bash
python scripts/evaluate/eval_detection.py --mode metric --batch-eval --ablation-dir outputs/ablation_study --stage all --weights-name best.pt --data data/processed/flir/dataset.yaml --img-size 640 --batch-size 16 --device 0 --workers 8 --save-csv --sort-by map5095
```

说明：当前数据配置只有 train 和 val，因此检测结果属于验证集结果。

## 四、跟踪评估

### 选择检测模型

优先从检测批量评估结果中选出少量候选模型，再进入跟踪测试。

### 执行跟踪测试

```bash
python scripts/evaluate/eval_tracking.py --weights outputs/ablation_study/ablation_exp07_eiou/weights/best.pt --data data/videos/thermal_test --tracker deepsort --output outputs/tracking/current --device 0 --img-size 640 --half --no-save-vid --no-save-txt --no-overlay
```

分别运行：

- deepsort
- bytetrack
- centertrack

## 五、结果汇总

### 检测结果绘图

```bash
python scripts/evaluate/plot_eval_summary.py --config configs/plot_eval_summary.yaml --input-dir outputs/detection/detection_eval_batch_xxx
```

### 跟踪结果绘图

```bash
python scripts/evaluate/plot_tracking_algorithm_comparison.py --config configs/plot_tracking_comparison.yaml
```

## 六、当前建议的实验闭环

1. 准备并核对数据集
2. 训练或整理已有检测权重
3. 批量跑检测评估
4. 按检测结果筛选候选模型
5. 对候选模型跑三种跟踪器
6. 汇总跟踪结果并确定最终方案

## 七、注意事项

- 不要把验证集结果写成独立测试集结论
- 跟踪测试前先固定检测模型，避免口径混乱
- 当前仓库不再维护部署链路，如需恢复，应单独分支重建
