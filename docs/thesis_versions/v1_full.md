# 基于改进YOLOv5的红外多目标检测与跟踪系统研究

---

## 摘要

针对红外图像纹理缺失、对比度低导致的目标检测与跟踪困难，本文以FLIR红外数据集为基础，围绕检测改进、跟踪选型与嵌入式部署三个环节开展系统性实证研究。在检测环节，设计13组严格控变消融实验，系统评估轻量化主干、注意力机制与损失函数优化三类改进策略的独立及组合效果，发现EIoU损失函数以零参数增量取得全消融最优精度mAP@0.5达0.817，并揭示组合改进策略在小模型条件下的负迁移效应。在跟踪环节，完成三种算法与五种检测模型的交叉对比，ByteTrack以ID Switch仅26取得最优轨迹连续性，实证验证纯运动关联策略在红外无纹理场景中优于外观特征方法。在部署环节，面向RV1126B嵌入式平台完成INT8量化与五项NPU优化，实现板端33FPS实时推理，经三轮参数迭代改善板端跟踪质量。

**关键词**：红外目标检测；YOLOv5；EIoU损失函数；消融实验；ByteTrack；RV1126B；嵌入式部署

---

## Abstract

Addressing the challenges of texture deficiency and low contrast in infrared images for object detection and tracking, this thesis conducts a systematic empirical study on the FLIR infrared dataset, covering detection improvement, tracker selection, and embedded deployment. In the detection stage, 13 strictly controlled ablation experiments evaluate three categories of improvements: backbone lightweighting, attention mechanisms, and loss function optimization. The EIoU loss function achieves the optimal accuracy (mAP@0.5=0.817) with zero additional parameters, while combined strategies exhibit negative transfer under small-model conditions. In the tracking stage, ByteTrack achieves the lowest ID Switch of 26 across cross-comparison tests, empirically validating the superiority of pure motion association over appearance-based methods in textureless infrared scenarios. For the RV1126B embedded platform, INT8 quantization and five NPU optimizations enable real-time inference at 33 FPS, with three rounds of on-board parameter tuning further improving tracking quality.

**Keywords**: Infrared Object Detection; YOLOv5; EIoU Loss; Ablation Study; ByteTrack; RV1126B; Embedded Deployment

## 第1章　绪论

### 1.1　研究背景与意义

红外成像技术利用物体发射的红外辐射进行成像，不依赖外界光照条件，在夜间、烟雾、低照度等可见光失效的环境下仍能有效工作，因此在安防监控、自动驾驶辅助、无人机巡检和工业检测等领域得到广泛应用[1][2]。需要特别说明的是，本文所使用的FLIR ADAS数据集中的红外图像是长波红外（LWIR, 8-14μm）探测器输出的灰度强度图——像素值与目标红外辐射强度正相关，高辐射物体（如人体、运行中的发动机）呈现为较亮的灰度值，低辐射背景呈现为较暗的灰度值——但图像本身不携带显式温度数据，也不是常见的伪彩色热力图。本文后续统一使用"红外图像"一词指代这类灰度红外强度图，以区别于携带温度信息的热红外成像。

红外图像与可见光图像在成像特征上存在本质区别。可见光图像反映物体对外界光源的反射特征，包含丰富的颜色信息（RGB三通道独立）和纹理细节（边缘清晰、表面图案可辨），这些丰富的视觉线索为深度学习特征提取提供了充足的输入信息。而红外图像反映物体自身的辐射强度，呈现为灰度图像（三通道值完全相同），缺乏颜色和纹理细节[3]。在这样的成像条件下，目标与背景之间仅存在微弱的灰度差异，目标边界呈渐变过渡（辐射强度从目标到背景平滑衰减）而非锐利边缘，且不同类别目标的外观高度相似（如行人和远处车辆在灰度图中都表现为亮色区域）。这些特性对深度学习目标检测和多目标跟踪算法提出了独特的挑战——源自可见光领域的改进策略是否在红外场景下同样有效，需要通过受控实验验证。

从工程应用角度看，端侧推理（Edge Inference）相比云端推理在延迟、带宽和隐私方面具有明确优势。端侧推理的端到端延迟约30-50ms（一次本地前向传播），而云端推理需要经过图像压缩、网络传输、服务器排队和结果回传，总延迟可达100-900ms，对于车辆避障等时间敏感应用不可接受。对于部署在偏远地区（如电力线路巡检、边境安防）的红外监控设备，有限的网络带宽使端侧推理成为唯一可行方案。此外，端侧推理的视频数据全程留在本地设备上，从源头消除了隐私泄露风险[5]。

以Rockchip RV1126B为代表的嵌入式AI芯片，集成了NPU（Neural Processing Unit）神经网络处理单元，在典型功耗仅2-3W的条件下提供3.0 TOPS的INT8推理算力[7]。与PC端GPU动辄100-300W的功耗相比，嵌入式NPU的功耗效率优势使其成为边缘端智能视觉系统的理想载体。因此，如何在嵌入式NPU平台上实现兼顾精度和速度的红外目标检测与跟踪，是一个具有明确工程价值的研究问题。

然而，从训练好的深度学习模型到嵌入式板端的实时推理系统，中间存在一条复杂的工程链路：模型需要经过ONNX格式导出、INT8量化转换、交叉编译、板端推理框架开发、NPU性能优化，以及因量化精度损失导致的跟踪参数重新适配等多个环节。现有文献通常只报告最终的推理速度数据，缺少从量化方案选择、逐项性能优化到板端参数适配的完整工程实践记录。本文将完整记录这一工程链路的每个关键决策和实验数据，为同类嵌入式视觉系统的开发提供可复现的参考。

### 1.2　国内外研究现状

目标检测领域经历了从传统手工特征到深度学习方法的跨越式发展。早期的目标检测主要依赖HOG（方向梯度直方图）、Haar等手工特征结合SVM分类器进行滑动窗口检测[8]。这类方法依赖人工设计的特征提取器，对复杂场景的泛化能力有限。2014年R-CNN[9]首次将深度学习引入检测任务，使用CNN自动提取特征并在候选区域上进行分类，mAP相比HOG-SVM提升了超过20个百分点。随后Fast R-CNN[10]通过共享特征图计算将速度提升数十倍，Faster R-CNN[11]进一步引入区域提议网络（RPN）消除了外部候选区域生成的依赖，建立了两阶段检测器的完整框架。然而，两阶段检测器的推理速度通常限于15-20 FPS，难以满足实时需求。

以YOLO（You Only Look Once）系列为代表的单阶段检测器将目标检测建模为端到端的回归问题，从根本上消除了区域提议步骤[12]。YOLOv1在每个网格直接预测边界框和类别概率，实现了45 FPS的推理速度但精度较低。YOLOv2引入Batch Normalization和Anchor机制提升了精度[13]。YOLOv3通过多尺度预测和残差网络进一步缩小了与两阶段检测器的精度差距[14]。YOLOv5[15]凭借CSP-DarkNet53主干、FPN+PAN双向特征融合和便捷的训练-导出工具链，成为工业界应用最广泛的检测框架之一，其YOLOv5s变体以7.02M参数量在精度与效率之间取得了良好平衡。后续YOLOv7[18]引入E-ELAN高效层聚合模块、YOLOv8[19]改用解耦检测头和Anchor-Free设计，在架构上持续优化。但YOLOv5的模块化设计和成熟的RKNN部署支持使其仍是嵌入式场景的主流选择。SSD[20]作为另一种代表性单阶段检测器采用多尺度特征图直接预测，在速度和精度之间提供了不同于YOLO的权衡方案。

在红外检测方面，研究者主要沿两条路径展开。多模态融合方法[16][17]通过可见光-红外特征互补提升检测精度。Li等人[16]提出同时进行检测和分割的多光谱行人检测方法。Zhang等人[17]设计了循环融合模块实现多光谱特征的迭代精化。但多模态方法依赖双传感器严格对齐，系统复杂度高，且可见光在夜间失效时退化为单模态。单模态改进方法直接在红外图像上优化检测器，包括损失函数替换（SIoU[40]针对角度优化、EIoU[41]解耦宽高惩罚）、轻量化主干引入（Ghost Module[21]利用特征冗余性降低参数量、ShuffleNet[22]通过通道混洗促进信息交互、MobileNet[23][24]采用深度可分离卷积）和注意力机制嵌入（SE-Net[25]学习通道权重、CBAM[26]联合通道和空间注意力、CoordAtt[27]保留位置信息的坐标注意力）等。然而，现有红外检测研究大多采用"提出单一改进+对比基线"的范式，在各自不同的训练配置下报告结果，缺乏在同一数据集和统一训练条件下对多种改进策略进行公平系统性对比的工作。更关键的是，现有研究几乎未探讨组合使用多种改进策略时是否产生协同增益还是负迁移效应。

多目标跟踪领域形成了以"检测-跟踪"范式为主流的技术路线[28]——先由检测器在每帧中独立检测目标，再由跟踪器将相邻帧的检测结果进行跨帧关联。SORT[29]利用卡尔曼滤波预测目标位置并通过匈牙利算法进行IoU匹配，以极简的设计实现了基本的多目标跟踪功能。DeepSORT[30]在SORT基础上引入Re-ID外观特征描述子，通过深度网络提取目标的外观嵌入向量，在关联时联合运动距离和外观余弦距离进行匹配。在可见光场景中，不同行人的衣着颜色和图案差异为Re-ID提供了有效的判别线索，DeepSORT在遮挡后的身份恢复方面表现突出。然而，其Re-ID网络通常基于可见光行人数据集（如Market-1501）训练，在红外灰度图像中不同行人的外观高度相似（均表现为灰度亮斑），Re-ID特征的区分能力面临严重退化——但这一推测尚缺少系统性的红外实验验证。ByteTrack[31]提出双阈值二次关联机制，第一轮匹配高分检测，第二轮用低分检测挽救遮挡导致的失配，完全基于IoU运动匹配，不依赖外观特征，计算开销极小。CenterTrack[32]采用中心偏移预测方法，在检测网络中直接回归帧间偏移量实现关联。在红外场景中，外观特征的系统性退化使依赖Re-ID的方法理论上失去核心优势，而纯运动关联方法可能更适合——但这一理论分析缺少在红外数据集上的多算法、多检测模型的系统性交叉验证。

嵌入式部署方面，将深度学习模型部署到边缘端NPU平台面临模型量化精度损失和计算资源受限两大挑战[33][38]。主流推理框架包括NVIDIA TensorRT[34]（面向GPU，不适用于ARM NPU）、Rockchip RKNN-Toolkit2[35]（面向RV1126B等Rockchip NPU，提供Python转换和C API推理）、腾讯NCNN[36]和阿里MNN[37]（面向ARM CPU，不直接利用NPU加速）。对于RV1126B目标平台，RKNN-Toolkit2是唯一官方支持的NPU推理框架。RKNN-Toolkit2提供三种训练后量化方案：普通量化（normal）基于min-max统计确定量化参数，速度最快但对异常分布敏感；KL散度量化通过最小化量化前后分布散度寻找最优截断阈值，精度更稳定；MMSE（最小均方误差）量化逐层搜索使量化误差最小的参数配置，理论精度最高但校准时间为KL散度的数十倍，工程中成本效益比不足。现有文献通常只报告最终推理速度数据，缺少从量化方案选择、逐项性能优化到板端跟踪参数适配的完整工程实践记录，这正是本文工程贡献的重点。

### 1.3　现有研究不足与本文创新点

综合国内外研究现状，可归纳出以下三方面不足：

**不足一：红外检测改进研究缺乏系统性对比框架**。现有工作多为"单一改进+基线对比"，且在各自不同的训练配置和数据集划分下进行实验，难以判断不同改进策略的相对优劣。更关键的是，缺少对组合改进策略效果的实证分析——在工程实践中，"是否应该同时使用多种改进策略"是一个重要但缺乏实验依据的决策问题。

**不足二：跟踪算法在红外场景下的对比研究不充分**。现有红外跟踪研究通常仅评估单一跟踪器在单一检测模型上的表现，缺少多算法、多检测精度水平的交叉对比。特别是"外观特征在红外灰度图中是否有效"这一核心问题，缺少系统性的实验验证。

**不足三：嵌入式部署的完整工程链路缺少可复现记录**。现有文献多关注模型架构创新，对部署工程细节（量化方案选择、NPU优化策略、量化环境下跟踪参数适配）缺少详细记录。

针对上述不足，本文提出以下创新点：

**创新点1：面向红外场景的系统性消融实验方法论**。与现有"单一改进+基线对比"的范式不同，本文构建了涵盖3大类、13种配置的消融实验框架，所有实验在统一数据划分、训练超参和评估流程下执行。这一系统性设计的核心价值在于：它不仅评估了每种改进策略的独立效果，更关键地揭示了不同策略之间的交互效应。实验发现的"组合改进在小模型条件下存在负迁移效应"这一结论，为实际工程中的模型改进决策提供了重要参考——在资源受限场景下，应优先选择简单有效的单项改进（如损失函数优化），而非盲目叠加多种策略。

需要坦诚说明的是，本文的13组消融实验采取的是"多策略横向对比"而非"单一策略纵向深入调优"的研究路线。传统意义上更"深入"的做法是选定某一改进方向（如EIoU损失函数），在此基础上进行超参数搜索（如EIoU中宽高惩罚权重的调节）、学习率调度优化（如cosine/warmup策略对比）、数据增强策略迭代（如Mosaic增强的概率调节）等精细调参。本文选择横向对比路线的考量是：在红外检测这一相对新兴的应用场景中，首先需要建立一个可靠的基准参考——哪类改进策略最有效、哪些策略组合存在兼容性问题——然后后续研究者才能在此基础上对最有潜力的方向进行纵向深入。本文的系统性对比实验正是提供了这样的基准参考，其中发现EIoU损失函数是最有效的改进方向，即为后续纵向深入指明了方向。

**创新点2：红外场景下跟踪算法的交叉验证与实证发现**。本文在5种检测模型×3种跟踪算法的全交叉实验（15组配置）中，首次在红外场景中系统验证了"纯运动关联优于外观辅助关联"这一重要结论。ByteTrack的ID Switch（26）远低于DeepSORT（128），表明在红外灰度图像中，基于可见光训练的Re-ID外观特征非但未能提升跟踪性能，反而因域迁移问题成为干扰因素。这一实证发现具有方法论意义：在选择红外场景的跟踪算法时，应优先考虑纯运动关联方法。

**创新点3：检测-跟踪-部署一体化优化链路**。本文不仅完成了从消融实验到嵌入式部署的完整链路，还在链路中发现并解决了多个跨环节的适配问题：EIoU训练模型与普通量化的激活分布适配问题、板端INT8量化环境下跟踪参数需要重新适配的问题（IoU匹配阈值、轨迹确认帧数、可视延迟帧数）。通过3轮控制变量迭代实验确定了板端最优参数配置，使唯一轨迹ID降低14%。

### 1.4　研究内容与论文组织结构

本文的研究内容围绕"红外目标检测改进→跟踪算法选型→嵌入式部署优化→系统集成"四个递进环节展开：

（1）**检测模型改进**：在YOLOv5s基线模型上设计并执行13组控变消融实验，覆盖Ghost Module和ShuffleNet两种轻量化方案，SE/CBAM和CoordAtt两种注意力机制，以及SIoU和EIoU两种损失函数的单独和组合效果，确定最优改进策略。

## 第2章　相关理论与技术基础

本章介绍本文研究涉及的理论基础和关键技术，为后续消融实验、跟踪对比和嵌入式部署提供必要的知识背景。

### 2.1　FLIR红外数据集

本文使用FLIR公司发布的FLIR ADAS数据集[4]作为实验基础。该数据集由车载长波红外（LWIR）摄像头在城市道路场景中采集，包含训练集和验证集共约10,000余张红外图像。

需要重点说明的是，这些红外图像是LWIR探测器（工作波段8-14μm）的灰度输出——每个像素记录的是物体在该波段的辐射强度值，而非温度数据。图像以灰度格式存储，外观上与普通灰度照片类似，但成像机理完全不同：可见光图像反映物体对光的反射，红外图像反映物体自身的辐射强度。在这些图像中，人体因体温高于环境而呈现为较亮的灰度区域，运行中的车辆发动机同样呈亮色，而温度与环境接近的物体（如路面、建筑墙面）则呈现为中等或偏暗的灰度。本文后续统一使用"红外图像"一词指代这类灰度红外强度图，以区别于携带显式温度信息的热红外成像。

本文从FLIR数据集中提取person和car两个核心类别（NC=2），使用`scripts/data/prepare_flir.py`脚本将原始标注格式转换为YOLOv5所需的标准格式，并生成`data/processed/flir/dataset.yaml`数据集配置文件。转换后训练集包含约8,000张图像，验证集包含约2,000张图像。

与可见光数据集相比，FLIR红外数据集具有以下特殊性：第一，图像为单通道灰度图（在输入网络前被复制为三通道），缺乏颜色和纹理信息，特征维度显著降低；第二，目标边界通常呈渐变过渡（辐射强度从目标到背景平滑衰减）而非可见光中的锐利边缘，这对边界框回归精度提出了更高要求；第三，person和car两类目标在灰度图中的外观差异较小——均表现为亮色区域——类间区分难度增大；第四，不同行人目标之间的外观高度相似（均为灰度亮斑，无衣着颜色差异），这对需要外观特征的跟踪算法（如DeepSORT）构成根本性挑战。这些特点决定了源自可见光领域的改进策略不可无条件迁移至红外场景，需要通过受控实验验证其在红外条件下的实际效果。

### 2.2　YOLOv5网络架构

YOLOv5[15]是单阶段目标检测框架，将检测建模为统一的回归问题，在单次前向传播中同时输出目标的位置、类别和置信度。相比两阶段检测器（如Faster R-CNN），单阶段检测器省去了区域提议步骤，在推理速度上具有明显优势，更适合实时应用场景。

YOLOv5的网络架构由三部分组成：

**Backbone（主干网络）**采用CSP-DarkNet53结构。CSP（Cross Stage Partial）的核心思想是将输入特征图在通道维度上分为两部分，一部分经过残差网络处理，另一部分直接跳过，最后拼接输出。这种设计在保持特征学习能力的同时减少了约一半的计算量。输入图像经过一系列Focus模块（4倍下采样的空间到通道重排）、CBS（Conv-BN-SiLU）模块和CSP Bottleneck模块逐步提取多层次特征，在P3（stride=8, 80×80）、P4（stride=16, 40×40）和P5（stride=32, 20×20）三个尺度上输出特征图。P3特征图分辨率最高，适合检测小目标（如远距离行人）；P5特征图感受野最大，适合检测大目标（如近距离车辆）。

**Neck（颈部网络）**采用FPN+PAN双向特征融合结构[45]。自顶向下的FPN（Feature Pyramid Network）路径将P5的高层语义信息逐步上采样并与P4、P3特征融合，增强浅层特征的语义表达能力，有利于小目标检测。自底向上的PAN（Path Aggregation Network）路径将P3的精细空间定位信息回传至P4和P5，增强深层特征的定位精度，有利于大目标的精确定位。这种双向融合结构使每个检测尺度同时拥有丰富的语义信息和精确的空间定位能力。

**Head（检测头）**在P3/P4/P5三个尺度上各设置3组Anchor（预定义尺寸的参考框），每个Anchor预测4个边界框偏移量（中心偏移$\Delta c_x, \Delta c_y$和宽高缩放$\Delta w, \Delta h$）、1个目标置信度和NC个类别概率。本文NC=2（person和car）。三个检测头合计产生80×80×3 + 40×40×3 + 20×20×3 = 25,200个候选框，经过置信度阈值过滤和NMS（非极大值抑制）去重后输出最终检测结果。

本文选择YOLOv5s作为基线模型，其参数量为7.02M，GFLOPs为15.77，PC端GPU推理速度约137 FPS。选择YOLOv5s的原因包括：（1）参数量适中，在RV1126B 2GB内存和3.0 TOPS算力约束下可实现实时推理；（2）YOLOv5的ONNX导出和RKNN转换链路已被Rockchip官方验证，算子兼容性良好；（3）YOLOv5提供n/s/m/l/x五个规模变体，本文选择s变体作为精度与效率的平衡点，后续如需进一步提升精度可升级至m变体。

### 2.3　IoU系列损失函数

目标检测的边界框回归损失函数直接影响检测器的定位精度。传统的$\ell_1$和$\ell_2$损失函数独立优化边界框的四个坐标参数，无法反映边界框的几何关系（如重叠面积、中心距离、宽高比）。IoU（交并比）系列损失函数直接以边界框的几何关系为优化目标，理论上更适合检测任务。YOLOv5默认使用CIoU损失[39]，本文在消融实验中额外评估了SIoU[40]和EIoU[41]两种改进方案。

**CIoU损失（基线默认）**在标准IoU损失的基础上同时引入中心点距离惩罚和宽高比一致性惩罚：

$$\mathcal{L}_{CIoU} = 1 - IoU + \frac{\rho^2(b, b^{gt})}{c^2} + \alpha v$$

其中$\rho(b, b^{gt})$为预测框与真实框中心点的欧氏距离，$c$为两框最小外接矩形的对角线长度，$v = \frac{4}{\pi^2}(\arctan\frac{w^{gt}}{h^{gt}} - \arctan\frac{w}{h})^2$衡量宽高比的一致性，$\alpha = v / (1 - IoU + v)$为自适应平衡系数。CIoU的关键特征是将宽度和高度通过比值（$w/h$）耦合在一起——它惩罚的是宽高比的差异，而非宽度和高度各自的绝对误差。

**SIoU损失（exp06使用）**在距离代价之外引入角度代价，约束预测框中心向真实框中心移动时的方向。角度代价鼓励预测框沿水平或垂直方向（而非对角线方向）向目标靠近，理论上可以加速收敛：

$$\mathcal{L}_{SIoU} = 1 - IoU + \frac{\Delta + \Omega}{2}$$

其中$\Delta$为角度代价，$\Omega$为距离代价。SIoU在精确率指标上通常有提升（方向约束使边界框定位更精准），但对综合mAP的提升幅度有限。

**EIoU损失（exp07使用，本文最优方案）**将CIoU的宽高比耦合惩罚拆解为独立的宽度惩罚和高度惩罚：

$$\mathcal{L}_{EIoU} = 1 - IoU + \frac{\rho^2(b, b^{gt})}{c^2} + \frac{\rho^2(w, w^{gt})}{C_w^2} + \frac{\rho^2(h, h^{gt})}{C_h^2}$$

其中第三项$\frac{(w - w^{gt})^2}{C_w^2}$独立惩罚宽度误差（$C_w$为最小外接矩形的宽度），第四项$\frac{(h - h^{gt})^2}{C_h^2}$独立惩罚高度误差（$C_h$为最小外接矩形的高度）。

EIoU相比CIoU的核心改进在于解决了"梯度模糊"问题。具体而言：当预测框与真实框的宽高比已接近正确但宽度和高度的绝对值仍有偏差时（例如真实框40×80像素，预测框30×60像素，宽高比均为0.5但尺寸偏小25%），CIoU的宽高比惩罚项$v \approx 0$，对宽度的梯度$\frac{\partial v}{\partial w} \approx 0$，无法驱动模型进一步修正宽度和高度的绝对值。而EIoU对宽度的梯度$\frac{2(w - w^{gt})}{C_w^2} = \frac{2(30-40)}{C_w^2}$始终非零，对高度同理。在红外图像中目标边界的辐射强度过渡是渐变式的，边界框的宽度和高度回归面临较大不确定性，EIoU的这种解耦惩罚能够为每个维度提供独立的优化梯度，对精确回归尤为重要。

三种损失函数均不改变模型的网络结构和参数量，仅影响训练阶段的梯度计算，因此是"零代价"的改进策略——推理阶段的计算量和内存占用完全不变。

### 2.4　轻量化模块

模型轻量化是降低推理计算量的直接手段，但通常以精度损失为代价。本文消融实验中使用了两种轻量化方案，此处介绍其核心原理和适用性。

**Ghost Module**[21]基于一个关键观察：标准卷积生成的特征图中存在大量冗余——许多特征图之间高度相似，可以通过廉价变换生成。Ghost Module将标准卷积分解为两步操作：首先使用通道数减半的标准卷积生成"基础"特征图，然后对基础特征图施加廉价线性变换（通常为深度可分离卷积）生成"幻影"特征图，两组特征拼接得到与标准卷积等价的输出维度。理论上参数量和FLOPs均降至约一半。在本文exp02中，Ghost Module将YOLOv5s参数量从7.02M降至4.89M（-30.3%），GFLOPs从15.77降至10.41（-34.0%），但mAP@0.5从0.809降至0.774（-3.5pp），精度损失明显。

**ShuffleNet Channel Shuffle**[22]的核心设计是通道分割与混洗。输入特征在通道维度上分为两个等宽分支：一个保持恒等映射（Identity），另一个执行深度可分离卷积+逐点卷积的轻量化变换。两个分支的输出在通道维度上拼接后，通过通道混洗（Channel Shuffle）操作促进跨组的信息交互。在本文exp03中，参数量降至5.22M（-25.6%），mAP@0.5为0.780（-2.9pp），精度损失略小于Ghost但轻量化效果也较弱。

值得注意的是，轻量化模块在PC端GPU上的参数量和FLOPs下降并不直接等价于嵌入式NPU上的速度提升。NPU的脉动阵列（Systolic Array）架构针对标准卷积的规则计算模式深度优化，MAC阵列利用率可达85-95%。深度可分离卷积的通道间并行性不足，MAC阵列利用率可能降至30-50%。此外，Ghost Module的两阶段运算引入了额外的中间特征图读写开销，在DDR内存带宽受限的嵌入式平台上这一开销不可忽略。因此本文在嵌入式部署时选择标准YOLOv5s架构（exp07 EIoU改进），避免了非标准算子在NPU上的兼容性和效率风险。

### 2.5　注意力机制

注意力机制通过自适应地调整特征图中不同通道或空间位置的权重，使网络聚焦于任务相关的重要特征。本文评估了两种注意力机制在红外检测场景中的效果。

**SE-Net通道注意力**[25]通过"压缩-激励"（Squeeze-and-Excitation）两步操作实现通道权重学习。压缩步通过全局平均池化将每个通道的空间信息压缩为单个标量，生成通道描述符。激励步将通道描述符输入一个两层全连接瓶颈网络（压缩比$r$=16），输出各通道的重要性权重（sigmoid归一化至[0,1]），用于对原始特征图进行通道级别的缩放。参数增量约$2C^2/r$，在YOLOv5s中新增约0.18M参数。

**CoordAtt坐标注意力**[27]针对SE-Net在全局平均池化中丢失位置信息的局限，分别沿水平和垂直两个方向执行全局平均池化，保留了精确的一维位置信息。水平池化将$H \times W$特征图压缩为$H \times 1$，垂直池化压缩为$1 \times W$。两个方向的位置描述符经过共享的1×1卷积编码后，分别生成水平和垂直方向的注意力权重，组合为二维空间注意力图。CoordAtt在保持轻量级参数开销的同时，能够精确感知目标在图像中的位置分布。

在本文的实验条件下（YOLOv5s小模型、红外灰度图像、两类目标），注意力机制的有效性受到两方面根本性约束。第一，红外图像的三通道值完全相同（灰度图复制为BGR三通道），经过卷积层处理后，通道间的特征差异性相比可见光场景（R/G/B通道携带不同颜色信息）大幅降低，通道注意力模块可学习到的"重要通道"与"冗余通道"之间的区分度有限。第二，YOLOv5s各阶段的通道宽度为32、64、128、256、512，远小于大模型（如ResNet-50的64、256、512、1024、2048）。SE模块的压缩比$r$=16意味着64通道的特征图压缩后仅剩4个神经元，全连接层能建模的通道间交互关系极为有限。实验结果确认了上述分析：exp04（SE/CBAM）mAP@0.5=0.784、exp05（CoordAtt）mAP@0.5=0.786，均低于基线0.809。

### 2.6　多目标跟踪算法

多目标跟踪（MOT, Multi-Object Tracking）在视频序列中同时维持多个目标的运动轨迹和身份标识。主流方法遵循"检测-跟踪"范式[28]——先由检测器在每帧中独立检测目标，再由跟踪器将相邻帧的检测结果进行跨帧关联，维持目标的唯一身份ID。

**卡尔曼滤波器**（Kalman Filter, KF）是跟踪中运动预测的核心组件[42]，通过线性动态模型融合模型预测与传感器观测，在噪声环境下递归估计目标的最优状态。本文采用8维状态向量$\mathbf{x} = [c_x, c_y, a, h, \dot{c}_x, \dot{c}_y, \dot{a}, \dot{h}]^T$（分别为边界框中心坐标、宽高比、高度及其变化速度），观测向量$\mathbf{z} = [c_x, c_y, a, h]^T$对应检测器输出。

KF的工作过程分为预测步和更新步。预测步基于匀速运动假设预测当前帧状态：$\hat{\mathbf{x}}_k = F\mathbf{x}_{k-1}$，$\hat{P}_k = FP_{k-1}F^T + Q$，其中$F$为状态转移矩阵，$P$为误差协方差，$Q$为过程噪声协方差。更新步融合预测与观测：$K_k = \hat{P}_k H^T (H\hat{P}_k H^T + R)^{-1}$，$\mathbf{x}_k = \hat{\mathbf{x}}_k + K_k(\mathbf{z}_k - H\hat{\mathbf{x}}_k)$，其中$H$为观测矩阵，$R$为观测噪声协方差，$K_k$为卡尔曼增益。

本文实现中，噪声参数采用基于目标高度$h$的自适应方案：位置噪声权重$\sigma_{\text{pos}} = 1/20$，速度噪声权重$\sigma_{\text{vel}} = 1/160$。过程噪声$Q_{\text{pos}} = (\sigma_{\text{pos}} \cdot h)^2$，观测噪声$R_{\text{pos}} = (\sigma_{\text{pos}} \cdot h)^2$，使噪声参数能自适应匹配不同尺度目标的运动特征。

**匈牙利算法**[43]在多目标跟踪中用于求解检测-轨迹最优匹配。构建代价矩阵$C_{ij} = 1 - \text{IoU}(\hat{B}_i, D_j)$（$\hat{B}_i$为KF预测框，$D_j$为检测框），在$O(n^3)$时间复杂度内求解全局最优匹配方案。本文典型场景中代价矩阵规模不超过30×30，匹配耗时不超过0.1ms。

**ByteTrack**[31]的核心创新是双阈值二次关联机制。第一轮将高置信度检测框（score ≥ high_thresh）与所有活跃轨迹进行IoU匹配，匹配成功的对执行KF更新。第二轮将第一轮中未匹配到高分检测的活跃轨迹与低置信度检测框（low_thresh ≤ score < high_thresh）进行二次IoU匹配。二次关联的关键意义在于：遮挡或远距离目标的检测置信度通常较低，传统方法直接丢弃这些低分检测会导致轨迹中断和ID切换，ByteTrack通过二次关联"挽救"这些低分检测，维持轨迹连续性。ByteTrack完全基于IoU运动匹配，不需要Re-ID外观特征网络，计算开销极小，特别适合嵌入式部署。

轨迹生命周期管理方面：新出现的高分检测创建tentative状态轨迹；连续匹配min_hits帧后升级为confirmed状态；连续max_age帧未匹配则删除。lost轨迹缓冲机制允许暂时丢失的轨迹在max_age帧内与新检测重新关联（重激活）。

**DeepSORT**[30]在SORT[29]基础上引入Re-ID（重识别）外观特征描述子。关联时联合运动距离（马氏距离或IoU距离）和外观余弦距离，加权决策匹配结果。Re-ID网络通常基于大规模可见光行人数据集（Market-1501等）预训练，提取128维外观嵌入向量。在可见光场景中，不同行人的衣着颜色、图案差异较大，Re-ID能够有效区分视觉上相似的目标，在遮挡后恢复正确的身份关联。然而，在红外灰度图像中，所有行人呈现为外观高度相似的灰度亮斑（无颜色差异、无纹理细节），Re-ID网络所依赖的核心判别线索完全缺失。从特征空间角度，可见光行人在Re-ID特征空间中呈清晰聚类（同人聚拢、异人分散），而红外行人的特征聚类结构严重退化，不同行人的余弦距离差异大幅缩小。

**CenterTrack**[32]采用中心偏移预测方法。在检测网络输入中添加前一帧图像和上一帧检测热力图作为额外通道，网络直接回归当前帧检测中心相对上一帧的偏移量$(\Delta x, \Delta y)$，利用偏移量实现帧间关联。CenterTrack将关联信息编码到检测网络内部，避免了显式外观特征的使用，但额外输入通道增加了网络的计算量。

在红外场景的理论分析中：运动状态（位置、速度、方向）是物理世界的属性，不受成像域变化的影响——无论可见光还是红外成像，目标的运动轨迹是相同的。因此纯运动关联策略（ByteTrack）理论上不受红外成像特征退化的影响。而外观特征（颜色、纹理）是成像域的属性，可见光到红外的域迁移直接导致外观特征失效。本文在第4章中通过系统实验验证了这一理论推断。

**【证据】** `src/tracking/kalman_filter.py`；`src/tracking/bytetrack_tracker.py`；`src/tracking/unified_tracker.py`

### 2.7　RKNN量化部署技术

RV1126B[44]是Rockchip面向智能视觉应用的系统级芯片（SoC），集成了四核Cortex-A53 CPU（主频1.5GHz）和一颗NPU（神经网络处理单元，INT8峰值算力3.0 TOPS），配备2GB LPDDR4内存，典型功耗约2-3W。NPU采用Rockchip自研的脉动阵列加速架构，内部512KB SRAM的访问延迟约1-2个时钟周期，远低于外部LPDDR4的约100个时钟周期。在推理过程中，合理利用SRAM缓存热点数据能够显著降低内存访问瓶颈。

**INT8量化原理**。量化是将浮点数模型转换为定点数模型的过程，目的是在可接受的精度损失范围内大幅降低模型的存储和计算开销。INT8量化将FP32浮点参数映射到INT8整数范围[-128, 127]：

$$x_{\text{int8}} = \text{round}(x / s) + \text{zp}$$

反量化过程为$\hat{x} = (x_{\text{int8}} - \text{zp}) \times s$，其中$s$为缩放因子（scale），zp为零点偏移（zero point）。量化步长$\Delta = (x_{\max} - x_{\min})/255$，均匀量化的量化噪声功率约为$\Delta^2/12$。INT8运算相比FP32运算在NPU上具有约4倍的吞吐量提升和约4倍的存储压缩。

RKNN-Toolkit2提供三种训练后量化（PTQ）方案，本文对其适用性进行了分析和选择：

**（1）普通量化（Normal / Asymmetric Affine）**。根据校准数据集的统计激活值最大最小值直接确定scale和zero_point参数：$s = (x_{\max} - x_{\min})/(2^8-1)$，$\text{zp} = -\text{round}(x_{\min}/s)$。优点是转换速度最快（约5分钟），适合激活分布较为均匀的模型。缺点是对长尾分布或异常值敏感——若校准数据中存在少量极端激活值，量化范围被不必要地拉大，导致大部分正常值的有效量化等级减少。

**（2）KL散度量化（KL Divergence）**。不假设激活分布形式，而是通过搜索使量化前后激活分布之间的Kullback-Leibler散度最小的截断阈值：

$$\text{KL}(P \| Q) = \sum_i P(i) \log\frac{P(i)}{Q(i)}$$

其中$P$为原始FP32激活分布，$Q$为量化后INT8值的分布。KL散度量化在分布空间中寻找使信息损失最小的量化映射。校准数据使用FLIR验证集随机采样的100张图像。转换时间约15分钟，量化精度通常优于普通量化。

**（3）MMSE量化（最小均方误差，Minimum Mean Square Error）**。逐层搜索使量化前后该层输出的均方误差最小的scale和zero_point配置。理论上精度最优，但校准过程需要逐层前向传播并搜索最优参数，时间复杂度远高于前两种方案（约4-6小时，为KL散度的数十倍）。在本文的实际测试中，KL散度量化与普通量化在最终检测精度上的差异已极小（详见第5章），MMSE的进一步精度改善空间有限而时间成本过高，因此本文最终未采用MMSE方案。

**EIoU训练模型与普通量化的适配说明**。EIoU损失通过独立惩罚宽度和高度误差，改变了模型训练过程中边界框回归分支的收敛路径和最终参数分布。相比CIoU训练的模型，EIoU模型的回归输出层激活值可能呈现更集中、更对称的分布形态（因为EIoU消除了CIoU的梯度模糊，使回归更加精确和收敛）。普通量化基于全局min-max统计进行线性映射，当激活分布高度集中时，min-max范围内的有效区间占比较小，量化效率降低。KL散度量化通过分布拟合自动确定截断阈值，能更好地适配这种特征。实际对比中KL散度量化的scale值更小、zp更接近零（详见5.2节表5-1），验证了上述分析。不过由于两种方案的最终检测精度差异极小，这一不适配问题在实际应用中不构成严重障碍。

**ONNX到RKNN转换链路**：首先使用YOLOv5的export.py脚本将PyTorch模型导出为ONNX格式（`--dynamic --opset 11`），采用3-branch输出方案——在ONNX导出时截断输出解码层，仅保留P3/P4/P5三个检测头的原始卷积输出。解码操作（sigmoid激活、网格坐标映射、Anchor映射、letterbox反变换等）移至板端C++代码中执行。这种设计确保了RKNN模型中仅包含NPU原生支持的标准卷积等算子，避免了非标准操作导致的NPU-CPU切换开销。

**【证据】** `deploy/rv1126b_yolov5/python/convert_yolov5_to_rknn.py`

### 2.8　本章小结

本章阐述了本文研究涉及的理论基础和技术背景。FLIR红外数据集提供了红外场景的标准化实验平台，其灰度成像特征决定了可见光改进策略需要经过验证才能迁移；YOLOv5s作为轻量级基线具有良好的嵌入式适配性；EIoU损失函数通过解耦宽高惩罚解决了CIoU的梯度模糊问题，是零代价的改进策略；ByteTrack的纯运动关联策略在外观特征退化的红外场景中具有理论优势；RKNN-Toolkit2提供了INT8量化部署的完整工具链，其中KL散度量化在精度和效率之间取得了最佳平衡，而MMSE方案因时间成本过高未被采用。

## 第3章　基于消融实验的检测模型改进

本章是全文的核心实验章节。通过设计并执行13组严格控变的消融实验，系统评估轻量化主干、注意力机制和损失函数优化三类改进策略在红外检测场景中的实际效果，包括各策略的独立贡献和组合效应。

### 3.1　改进思路与实验设计

本文的改进思路源于一个基本问题：在可见光领域行之有效的YOLOv5改进策略，在红外图像中是否同样有效？红外图像缺乏颜色和纹理信息的特殊性，使得这一问题的答案并不显然。现有文献中大多采用"提出单一改进+对比基线"的范式，且通常在各自不同的训练配置下进行实验，难以判断不同改进策略之间的相对优劣，更无法分析策略间的交互效应。

为系统性地回答上述问题，本文采用严格控变的消融实验设计。所有13组实验使用完全相同的训练配置文件（`configs/ablation/train_profile_controlled.yaml`），包括统一的数据集划分（FLIR train/val）、训练轮数（100 epochs）、批大小（batch=16）、初始学习率（lr0=0.01）、学习率调度（余弦退火）和数据增强策略（Mosaic、HSV增强、随机翻转等）。每组实验仅改变一个（或指定的几个）实验变量，其余保持完全一致。这种控变设计确保了实验结论的可归因性——观察到的性能差异可以明确归因于改变的实验变量，而非训练配置的差异。

实验评估指标统一使用YOLOv5 val.py在FLIR验证集上的标准评估流程，报告的核心指标包括：Precision（精确率）、Recall（召回率）、mAP@0.5（IoU阈值0.5下的平均精度均值）和mAP@0.5:0.95（IoU阈值0.5至0.95步长0.05的平均精度均值）。其中mAP@0.5是PASCAL VOC标准的主要指标，mAP@0.5:0.95是COCO标准的更严格指标。

### 3.2　基线模型与单项改进实验

**基线模型（exp01）**。使用默认CIoU损失训练的YOLOv5s模型，参数量7.02M，GFLOPs 15.77，PC端GPU推理速度137.68 FPS。在FLIR验证集上的检测结果为：Precision=0.859、Recall=0.710、mAP@0.5=0.809、mAP@0.5:0.95=0.514。基线模型的person类别mAP@0.5:0.95=0.426，car类别mAP@0.5:0.95=0.602，car精度显著高于person。这一类间差异与FLIR数据集的固有特征有关：car目标通常占据更大的像素面积且辐射特征稳定，而person目标尤其是远距离行人可能仅占20-30像素高度。

**轻量化改进实验**。exp02（Ghost Module）通过将标准CBS卷积替换为Ghost Module，实现了参数量30.3%的降幅（7.02M→4.89M），GFLOPs降低34.0%（15.77→10.41）。然而mAP@0.5从0.809降至0.774，下降3.5个百分点，Recall从0.710显著降至0.670（-4.0pp），说明在红外低对比度条件下过度压缩特征通道导致了关键信息丢失，使检测器更容易遗漏弱目标。exp03（ShuffleNet）将参数量降至5.22M（-25.6%），GFLOPs降至11.24（-28.7%），mAP@0.5为0.780（-2.9pp），Recall为0.672（-3.8pp），精度损失略小于Ghost。

两组轻量化实验的精度损失幅度（2.9-3.5pp）远超训练随机性范围（±0.3pp），说明轻量化带来的信息容量不足是确定性的、结构性的问题。在红外图像本就缺乏颜色和纹理等高维特征信息的条件下，进一步压缩模型的特征通道宽度对检测精度的影响比可见光场景更为敏感。PC端推理速度方面，两组轻量化模型的FPS反而低于基线（110 vs 138 FPS），这是因为PyTorch在GPU上对标准卷积有高度优化的cuDNN内核，深度可分离卷积的优化程度不足以弥补FLOPs的减少。

**注意力机制改进实验**。exp04（SE/CBAM）在C3模块内P4和P5特征层后插入注意力模块，参数量增至7.20M（+0.18M），mAP@0.5降至0.784（-2.5pp），Recall降至0.681（-2.9pp）。这一反直觉的结果——引入注意力机制反而降低了精度——可从以下角度理解：

第一，红外灰度图的三通道值完全相同，通道维度的特征差异性弱于可见光，SE的通道权重学习缺乏足够的区分信号，可能学习到接近均匀的无效权重，而引入的额外参数和计算反而增加了训练的不稳定性。

第二，YOLOv5s的通道宽度有限（32-512），SE模块的压缩比$r$=16导致64通道压缩后仅4个神经元，全连接层能建模的通道间交互关系极为有限。可以类比为：让4个"代表"去总结64个通道的重要性，信息压缩过于极端。

第三，FLIR数据集仅含person和car两类目标，类别少时注意力机制难以发挥其类间区分优势。

### 3.3　损失函数改进实验

损失函数改进是一种"零参数代价"的优化策略——仅修改训练阶段的损失计算方式，不改变推理阶段的网络结构和计算量。这一特性使损失函数改进在嵌入式部署场景中具有特殊价值：检测精度的提升不需要消耗额外的NPU算力。

**SIoU损失（exp06）**。超参配置：`configs/ablation/hyp_siou_only.yaml`，`iou_type: siou`。参数量和GFLOPs与基线完全一致（7.02M / 15.77），确认损失函数改进不影响网络结构。实验结果：Precision=0.871（全部13组实验最高，较基线+1.2pp），Recall=0.705（-0.5pp），mAP@0.5=0.811（+0.2pp），mAP@0.5:0.95=0.512（-0.2pp）。PC端推理速度133.25 FPS。

SIoU在精确率上的显著提升表明角度约束确实使边界框定位更精准、减少了误检。但Recall的下降说明SIoU使检测器变得更"保守"——对检测结果的边界框质量要求更高，部分弱目标因边界框不够精确而被抑制。mAP@0.5仅提升0.2pp，mAP@0.5:0.95反而略有下降，说明SIoU的角度约束对综合检测性能的改善有限。

**EIoU损失（exp07）——全消融最优方案**。超参配置：`configs/ablation/hyp_eiou_only.yaml`，`iou_type: eiou`。参数量和GFLOPs同样与基线一致。

实验结果：Precision=0.859（与基线持平）、Recall=0.719（+0.9pp，全部13组实验最高）、mAP@0.5=0.817（+0.85pp，全部13组实验最高）、mAP@0.5:0.95=0.516（+0.2pp，全部实验最高）。PC端推理速度153.08 FPS。

EIoU的各项分析如下：

**Recall改善的重要性**。EIoU将Recall从0.710提升至0.719（+0.9pp），这是全部13组实验中唯一显著提升Recall的方案。Recall反映检测器发现目标的能力，提升Recall意味着减少了漏检。在安防和驾驶辅助场景中，漏检一个行人的后果远比误报一个非目标严重。EIoU通过解耦的宽高惩罚提供更精确的边界框回归，使部分原本因边界定位不准确而在NMS后处理中被淘汰的弱检测得以保留。

**EIoU优势的机制解释**。CIoU将宽度和高度通过比值耦合，当预测框的宽高比已接近真实值但绝对尺寸仍有偏差时（如真实框40×80，预测框30×60，宽高比均为0.5但尺寸偏小25%），CIoU的惩罚项$v \approx 0$，无法驱动进一步修正。EIoU的宽度惩罚梯度$\propto (30-40)=-10$、高度惩罚梯度$\propto (60-80)=-20$，分别独立驱动两个维度的修正。在红外图像中目标边界的辐射强度是渐变过渡的，检测器在回归宽度和高度时面临较大不确定性，EIoU的这种解耦惩罚为每个维度提供了独立的优化信号。

**训练过程特征**。exp07的box_loss初始值约0.10（高于exp01的0.08），因为EIoU的三个独立惩罚项（中心距离+宽度误差+高度误差）在训练初期产生了更大的损失值。但box_loss在训练中期（第30-60轮）的下降速度明显快于exp01，最终收敛至约0.025（低于exp01的0.03），说明EIoU为模型提供了更清晰的优化方向。obj_loss和cls_loss与exp01几乎一致，确认EIoU的改进仅影响边界框回归分支，不影响置信度预测和类别分类。best.pt权重在第92轮更新（exp01为第87轮），说明EIoU有更大的后期改善潜力。

**分类详细指标**：person mAP@0.5:0.95=0.432，car mAP@0.5:0.95=0.600。car精度显著高于person，与基线模型的类间差距保持一致——这是FLIR数据集目标尺寸和辐射特征差异的固有结果。

**EIoU vs. SIoU对比**。两者均为零代价损失函数改进，但优化方向不同：SIoU侧重角度约束，提升精确率但降低召回率，mAP变化不大；EIoU侧重尺寸约束（解耦宽高），在精确率不变的前提下显著提升召回率，mAP取得更大改善。对于红外目标检测而言，EIoU的优势更加契合实际需求。

**核心发现——负迁移效应**。所有6组组合实验的mAP均低于单项最优（exp07的0.817），且除exp09外均低于基线（0.809）。这一结果揭示了一个重要发现：在本文的实验条件下（YOLOv5s小模型+FLIR红外数据集+两类目标），多种改进策略的叠加并不能产生协同增益，反而因策略间的相互干扰导致精度下降。

**各组合实验分析**：

**exp08（Ghost+注意力）** mAP=0.754，是全部13组中最低。比单独Ghost（0.774）低2.0pp，比单独注意力（0.784）低3.0pp。机制分析：Ghost Module将通道数减半，SE注意力的通道压缩-激励操作在更窄的通道上运行（如32通道压缩至仅2个神经元），瓶颈层信息量进一步减少，产生了双重信息损失的叠加效应。PC端FPS仅102.31，也是全部实验最低。

**exp09（Ghost+EIoU）** mAP=0.790，是组合实验中的亮点。相比单独Ghost（0.774）提升1.6pp，说明EIoU的优化效果具有泛化性——即使模型特征容量因轻量化而不足，解耦的宽高惩罚仍能为回归提供有效引导。exp09在参数量减少30.3%的前提下mAP仅损失1.9pp（vs基线），展现了良好的精度-效率平衡。对于NPU算力极度受限或需在单个NPU上并行运行多模型的场景，exp09提供了一个可接受的轻量化选项。

**exp10（注意力+EIoU）** mAP=0.798。相比单独EIoU（0.817）下降1.9pp，说明注意力模块的引入"干扰"了EIoU已优化的特征表达。分析：注意力模块通过通道重加权改变了特征图的分布，EIoU损失在训练过程中已基于标准特征分布学习了最优的宽高回归策略，注意力重加权后的特征分布偏离了EIoU适配的最优分布，两者的优化目标产生了轻微冲突。

**exp11-13 ShuffleNet组合递进分析**：

| 实验 | 改进叠加 | mAP@0.5 | 较exp03增量 |
|------|---------|---------|------------|
| exp03 | Shuffle only | 0.780 | — |
| exp11 | Shuffle+CoordAtt | 0.763 | -1.7pp |
| exp12 | Shuffle+CoordAtt+SIoU | 0.767 | -1.3pp |
| exp13 | Shuffle+CoordAtt+EIoU | 0.768 | -1.2pp |

CoordAtt引入使mAP下降1.7pp（exp11 vs exp03），ShuffleNet的通道分割使每个分支仅拥有一半通道宽度，CoordAtt在窄通道上的位置编码能力受限。SIoU和EIoU损失在此基础上分别回升0.4pp和0.5pp，但效果被轻量化和注意力的负面影响严重压缩——在单独实验中EIoU比SIoU高0.6pp（0.817 vs 0.811），在三重组合中仅高0.1pp（0.768 vs 0.767）。

**统计显著性分析**。EIoU vs Baseline的差距（0.85pp）超过了典型训练随机性范围（±0.3pp），但不构成压倒性优势。更关键的是EIoU在Recall维度的改善（+0.9pp）方向一致且与损失函数设计的理论预期吻合。轻量化实验的精度损失（2.9-3.5pp）和组合实验的负迁移（exp08 -5.5pp vs基线）远超随机波动范围，结论具有高度确定性。

**三大关键发现**：

（1）**损失函数改进是投入产出比最优的策略**。exp07 EIoU以零参数代价取得全消融最优精度，对嵌入式部署意味着检测精度的提升无需消耗额外NPU算力。这一发现的工程指导意义在于：在嵌入式场景下，优先优化损失函数是提升检测精度的最佳策略。

（2）**Ghost+EIoU是轻量化最佳组合**。exp09在参数量减少30.3%条件下mAP仅损失1.9pp，适合NPU算力极度受限或多模型并行场景。

（3）**组合改进在小模型+红外数据条件下存在负迁移效应**。exp08到exp13的结果一致表明策略间存在系统性的相互干扰。这一发现挑战了"多种改进叠加必然更好"的直觉假设，为实际工程中的模型改进决策提供了重要参考——应优先选择简单有效的单项改进，而非盲目叠加。

### 3.5　本章小结

本章通过13组严格控变消融实验系统评估了三类改进策略在红外检测中的效果：

（1）**EIoU损失函数是最优策略**：在不增加参数量的前提下取得全消融最优精度mAP@0.5=0.817（+0.85pp vs Baseline），其解耦的宽高惩罚有效改善了红外图像中边界模糊目标的回归精度。

（2）**轻量化改进以确定性精度损失为代价**：Ghost和ShuffleNet分别降低参数量30.3%和25.6%，但mAP分别下降3.5pp和2.9pp。

（3）**注意力机制在小模型+红外数据条件下无正向增益**：SE/CBAM和CoordAtt均未超越基线。

（4）**多策略组合存在负迁移效应**：6组组合实验的精度均低于单项最优。

基于上述结论，本文选择exp07（EIoU损失函数）作为后续跟踪和部署的主线模型，保留exp09（Ghost+EIoU）作为轻量化备选。

## 第4章　多目标跟踪算法对比实验

在确定EIoU损失函数为最优检测模型改进后，本章进一步探讨红外视频场景中的多目标跟踪问题。通过系统性交叉对比实验，评估三种代表性跟踪算法在红外场景中的表现，确定最适合红外检测-跟踪链路的算法组合。

### 4.1　跟踪实验设计与评估指标

**测试视频选取**。从FLIR红外数据中选取两个代表性视频序列：seq006（221帧，行人车辆混合交通，车辆存在快速横向运动，帧间位移可达20像素）和seq009（565帧，行人密集场景，包含5-8个行人在人行道上的混合运动，频繁遮挡和交叉路径）。两个序列分别评估跟踪器对快速运动和密集遮挡两种典型挑战的应对能力。

**检测模型选取**。选取5个代表性检测模型作为上游输入，覆盖从高精度到低精度的完整范围：

| 模型 | mAP@0.5 | 精度定位 |
|------|---------|---------|
| exp07_eiou | 0.817 | 全消融最优 |
| exp01_baseline | 0.809 | 标准基线 |
| exp09_ghost_eiou | 0.790 | 轻量化最优 |
| exp02_ghost | 0.774 | 单独轻量化 |
| exp12_shuffle_coord_siou | 0.767 | 三重组合（最低精度代表） |

选取5种不同精度水平检测模型的目的是验证跟踪算法的排序结论是否对检测精度具有鲁棒性——即是否存在某种跟踪算法仅在高精度检测下表现好、在低精度下反而不如其他算法的情况。

**跟踪算法选取**。ByteTrack（纯运动IoU关联）、DeepSORT（外观+运动混合关联）和CenterTrack（中心偏移预测关联）。三者在关联策略上存在本质区别，形成了理想的对照组。

**评估指标**。本文采用以下指标组合：

（1）**ID Switch Proxy**。通过统计输出轨迹ID的更新频次估算ID切换次数。每当某个目标被分配到与前一帧不同的ID时计为一次ID切换。数值越低表示轨迹连续性越好——跟踪器能够在遮挡、快速运动等干扰下维持目标的正确身份标识。

（2）**Unique ID Count**。整个视频序列中出现过的不同轨迹ID总数。在理想情况下，Unique ID Count应等于视频中实际出现的目标个数。数值越高说明产生了更多短命轨迹（目标被反复分配新ID），反映跟踪器的轨迹维持能力较差。

（3）**Match Rate**。以ByteTrack为例，定义为第一轮高分关联的匹配成功率（成功匹配的对数/总活跃轨迹数）。数值越高表示检测框与已有轨迹的IoU匹配越稳定。

（4）**跟踪FPS**。包含检测+跟踪全流程的端到端帧率，评估算法的实时性。

需要说明的是，本文使用的ID Switch Proxy是基于跟踪器输出的代理统计指标，并非基于真实轨迹标注的标准CLEAR MOT IDSW指标（FLIR测试序列缺乏逐帧的真实轨迹ID标注）。因此本文的跟踪指标侧重于算法间的相对对比排序，而非与其他论文的绝对精度比较。但由于三种算法均在完全相同的检测输入和测试视频上评估，相对排序结论是可靠的。

**【证据】** `configs/tracking_config.yaml`；`scripts/evaluate/eval_tracking.py`

**核心发现1：ByteTrack在所有检测模型上均取得最低ID Switch**。以exp07为检测源时ByteTrack ID Switch=26，DeepSORT=128（4.9倍），CenterTrack=87（3.3倍）。ByteTrack的优势不仅绝对值领先，更关键的是这一领先跨越了所有5种检测精度水平保持一致——在最低精度的exp12检测输入下，ByteTrack的ID Switch仍为52，低于DeepSORT在最高精度exp07下的128。这一结论的鲁棒性排除了"ByteTrack仅在高精度检测下领先"的可能性。

**核心发现2：DeepSORT的Re-ID在红外域完全失效**。DeepSORT的ID Switch在所有15组配置中均为最高（128-183），不仅远高于不使用外观特征的ByteTrack（26-52），甚至显著高于CenterTrack（87-119）。DeepSORT本应通过Re-ID外观特征在遮挡后重新识别目标（相比纯运动方法具有优势），但在红外灰度图中反而成为所有算法中表现最差的。

这一反常结果直接证实了2.6节的理论分析：DeepSORT的Re-ID网络基于可见光行人数据集训练，学习到的外观嵌入空间对颜色、纹理等可见光特有线索高度敏感。在红外灰度图中，所有行人呈现为无纹理差异的灰度亮斑，Re-ID网络输出的外观距离失去了区分不同个体的能力。更严重的是，错误的外观距离（将不同行人误判为"外观相似"或将同一行人判为"外观不同"）直接干扰了关联决策，导致大量错误的ID切换。这不是"Re-ID没有帮助"的问题，而是"Re-ID在红外域产生了负面干扰"。

**核心发现3：检测精度直接影响跟踪质量**。以ByteTrack为标杆，在不同检测模型上的ID Switch呈现与mAP的单调负相关关系：

| 检测模型 | mAP@0.5 | ByteTrack ID Switch |
|---------|---------|-------------------|
| exp07_eiou | 0.817 | 26 |
| exp01_baseline | 0.809 | 31 |
| exp09_ghost_eiou | 0.790 | 38 |
| exp02_ghost | 0.774 | 45 |
| exp12_s_c_siou | 0.767 | 52 |

检测精度每下降约5pp，ID Switch增加约13次。机制：更优的检测精度意味着边界框的位置和尺寸更准确，帧间IoU计算更稳定，关联成功率更高。特别是EIoU优化的边界框在帧间具有更好的尺度一致性（因为宽度和高度被独立优化，避免了CIoU的梯度模糊导致的尺度波动），这直接提升了IoU关联的稳定性。检测改进的收益通过跟踪链路得到了放大。

**核心发现4：ByteTrack的计算效率最优**。ByteTrack的跟踪FPS（44-47 FPS）高于DeepSORT（31-34 FPS）和CenterTrack（28-30 FPS）。ByteTrack的跟踪环节仅包含KF预测（O(n)）和匈牙利匹配（O(n³)，实际n<30），不需要额外的Re-ID特征提取网络（DeepSORT额外消耗约5ms/帧）或帧间特征编码网络（CenterTrack额外消耗约8ms/帧）。这一特性使ByteTrack特别适合算力受限的嵌入式平台。

**ByteTrack的Match Rate分析**。exp07+ByteTrack的Match Rate为0.839，意味着每帧约84%的活跃轨迹在第一轮高分关联中即成功匹配。剩余16%的轨迹进入第二轮低分关联，其中一部分成功通过低分检测框匹配（ByteTrack的核心优势场景），另一部分因确实无匹配而进入lost状态。Match Rate随检测精度的下降而降低（exp12仅0.791），说明检测框质量是关联成功率的基础保障。

### 4.3　卡尔曼滤波参数优化

卡尔曼滤波器的噪声参数直接影响跟踪框的平滑度和跟随性，是跟踪质量的关键调节旋钮。

过程噪声$Q$控制模型对目标运动变化的响应灵敏度。$Q$过大时滤波器过于信任观测（跟踪框随检测框逐帧跳动，视觉上"抖动"）；$Q$过小时滤波器过于信任匀速运动预测（跟踪框对突然变速或变向的响应迟缓，视觉上"延迟跟随"）。观测噪声$R$控制对检测结果的信任程度，$R$过小时任何检测噪声都被全量接受，$R$过大时检测结果被过度平滑。

在红外场景中，目标边界模糊导致检测框位置在帧间存在较大波动（即使目标本身运动平稳），因此$Q$和$R$的设置需要特别谨慎。本文采用基于目标高度$h$的自适应方案：$Q_{\text{pos}} = (\sigma_{\text{pos}} \cdot h)^2$，$R_{\text{pos}} = (\sigma_{\text{pos}} \cdot h)^2$，其中$\sigma_{\text{pos}} = 1/20$、$\sigma_{\text{vel}} = 1/160$。$R = Q$（过程噪声和观测噪声相等）使卡尔曼滤波在预测和观测之间取得对称平衡——既不过度信任可能抖动的检测框，也不过度依赖匀速运动假设。

板端C++实现中，KF参数与PC端Python实现（`src/tracking/kalman_filter.py`）严格对齐。在初期板端测试中发现跟踪框出现明显的延迟跟随现象——快速横向运动的车辆，跟踪框始终落后于实际检测框约2-3帧。排查后发现是KF初始化时噪声参数未正确设置（$Q=0, R=0$导致KF的卡尔曼增益$K \approx 0$，完全不更新状态，退化为纯预测模式）。修复$\sigma_{\text{pos}} = 1/20$后，快速运动车辆的跟踪框能够及时追上运动趋势，延迟现象消除。

此外，板端实现中采用了输出平滑策略：跟踪框的中心坐标使用KF滤波值（KF通过速度分量预测下一帧位置，对运动目标的跟踪准确且响应快），宽高尺寸使用EMA（指数移动平均）平滑值（$\alpha_{\text{ema}} = 0.3$），避免宽高在帧间的突然跳变（红外目标边界模糊导致检测框尺寸在帧间波动较大）。两者结合实现了"位置跟踪准确+尺寸显示稳定"的视觉效果。

### 4.4　本章小结

本章通过5模型×3跟踪器的全交叉对比实验（15组配置），在红外视频场景中系统比较了三种跟踪算法的性能：

（1）**ByteTrack是红外场景的最优跟踪器**：ID Switch仅26（exp07检测源），分别为DeepSORT的1/5和CenterTrack的1/3.3，且跟踪FPS最高。

（2）**纯运动关联策略在红外域优于外观辅助方法**：ByteTrack的全面领先验证了运动状态是跨成像域不变的物理属性这一理论分析。

（3）**Re-ID外观特征在红外灰度图中完全失效**：DeepSORT的表现最差不是因为"Re-ID没帮忙"，而是Re-ID产生了"负面干扰"。

（4）**检测精度通过跟踪链路放大**：EIoU优化的检测框带来更稳定的帧间IoU，直接降低了ID Switch。

（5）**KF噪声参数对跟踪平滑性至关重要**：$\sigma_{\text{pos}} = 1/20$、$R = Q$为最优平衡。

基于上述结论，选择**exp07_eiou + ByteTrack**作为系统主线方案，用于后续嵌入式部署。

### 5.1　部署方案总体设计

端到端部署链路包含四个环节，涉及三台设备之间的协作：

**环节1：PC端模型导出**。在训练服务器（GPU机器）上将PyTorch最优权重导出为ONNX格式。采用3-branch输出方案——在ONNX导出时截断YOLOv5的输出解码层，仅保留P3/P4/P5三个检测头的原始卷积输出（每个branch输出形状为[1, 21, H, W]，21=3×(4+1+NC)，NC=2）。解码操作（sigmoid激活、网格坐标映射、Anchor尺寸映射、letterbox反变换等）移至板端C++代码中执行。这种设计确保了RKNN模型中仅含NPU原生支持的标准卷积、BN、激活等算子，避免了sigmoid等非标准操作在NPU上的低效执行（NPU执行sigmoid需回退至CPU，引入~2ms的上下文切换开销）。

**环节2：RKNN模型转换**。在PC端使用RKNN-Toolkit2 Python API将ONNX模型转换为RKNN INT8格式。转换过程包括模型加载、校准数据集配置和量化执行。校准数据使用FLIR验证集随机采样的100张红外图像。

**环节3：Ubuntu交叉编译**。在Ubuntu服务器上使用ARM GCC工具链交叉编译C++推理框架，生成可在RV1126B ARM架构上运行的可执行文件`bishe_rknn_video`。编译脚本`deploy/rv1126b_yolov5/build_rv1126b.sh`自动化了CMake配置、编译和打包流程。

**环节4：RV1126B板端推理**。通过ADB将可执行文件和RKNN模型推送至板端`/userdata/`目录，执行推理命令处理红外视频序列，输出含跟踪框和HUD信息的MP4视频文件。

### 5.2　模型量化转换

RKNN-Toolkit2提供三种训练后量化（PTQ）方案。本文对普通量化和KL散度量化进行了实验对比，并记录了MMSE量化的排除理由。

**普通量化（Normal / Asymmetric Affine）**：根据校准数据集上各层激活值的统计最大最小值直接计算量化参数$s$和$\text{zp}$。实现简单，转换速度最快（约5分钟），适合激活分布较为均匀的模型。但对长尾分布或异常值敏感——若校准数据中存在少量极端激活值，整个层的量化范围被不必要地拉大，大部分正常值只能使用INT8的一小部分有效等级（如256个等级中仅使用100个），量化精度降低。

**KL散度量化**：通过搜索使量化前后激活分布之间的Kullback-Leibler散度最小的截断阈值来确定量化参数。KL散度量化在分布空间中寻找使信息损失最小的量化映射，能够自动处理长尾分布——截断少量极端值以换取大部分正常值的更细粒度量化。校准数据同样使用FLIR验证集随机采样的100张图像。转换时间约15分钟。

**MMSE量化（未采用）**：逐层搜索使量化前后该层输出的均方误差$\text{MSE} = \frac{1}{n}\sum(x_i - \hat{x}_i)^2$最小的scale和zero_point配置。理论上精度最优，但存在两个工程问题：（1）校准时间约4-6小时，为KL散度的20-40倍，迭代实验成本过高；（2）在本文实际测试中，KL散度量化和普通量化的检测精度差异已极小（见下表），MMSE的进一步改善空间有限。权衡时间成本与精度收益后，本文未采用MMSE方案。

**【表5-1】** 量化方案对比

从表5-1可以观察到：KL散度量化的scale值显著小于普通量化（如第一个branch: 0.333 vs 0.787），zero_point也更接近零（81 vs 108）。更小的scale意味着更细的量化步长，更接近零的zp意味着零值映射更精确。这些参数特征说明KL散度量化确定了更紧凑的截断范围，对激活值的有效区间进行了更细粒度的量化。

然而在端到端检测数量对比中，两种方案的差异极小。这说明YOLOv5s模型的激活分布本身较为集中（没有严重的长尾现象），普通量化的min-max映射已足够胜任，KL散度量化的分布拟合优势未能转化为可感知的检测精度提升。

**EIoU训练模型与普通量化的适配分析**。EIoU损失通过独立惩罚宽度和高度误差，改变了模型训练过程中边界框回归分支（检测头最后一层卷积的前4个通道）的收敛路径。相比CIoU训练的模型，EIoU模型的回归输出可能呈现不同的激活值分布特征——EIoU消除了CIoU的梯度模糊问题，使回归更加精确收敛，激活值分布可能更集中和对称。普通量化对所有层统一使用min-max映射，当回归分支的激活分布偏离均匀假设时（如高度集中在某个窄范围内），有效量化等级的利用率降低。KL散度量化能够自适应地为每层确定最优截断阈值，对这种集中分布的适配性更好。

在实际部署中，由于两种方案的最终检测精度差异在噪声范围内，本文的板端实验主要基于普通量化方案（转换速度优势明显，5分钟 vs 15分钟），在需要最大化板端精度的场景下可选用KL散度量化作为稳妥选择。

**3-branch输出解码流程**。RKNN模型输出为3个branch的INT8张量，板端解码流程如下：

（1）`rknn_outputs_get`获取INT8原始数据，设置`want_float=0`避免RKNN框架内置的反量化操作（RKNN内置反量化未做SIMD优化，效率低于自实现版本）。

（2）使用NEON SIMD指令加速反量化为FP32：对每4个INT8值并行执行$\hat{x}_i = (x_i^{\text{int8}} - \text{zp}) \times s$，理论吞吐量为标量版本的4倍。

（3）**logit预滤波**——这是后处理中收益最大的优化。标准流程对每个Anchor先执行sigmoid得到目标置信度再与阈值比较，但sigmoid涉及指数运算，计算开销大。logit预滤波利用sigmoid的单调性和反函数：$\text{logit}(p) = \ln\frac{p}{1-p}$，将阈值0.25映射到logit空间得到$\text{logit}(0.25) \approx -1.099$，直接在反量化后的FP32值上与-1.099比较。由于约98%的Anchor为背景（置信度远低于阈值），logit预滤波跳过了这98%候选的sigmoid计算和后续解码，仅对通过预滤波的约2-3%候选执行完整的sigmoid+解码+NMS。

（4）对通过预滤波的候选执行完整解码：sigmoid激活得到obj_conf和class_prob、边界框中心和尺寸解码（$b_x = 2\sigma(\Delta c_x) - 0.5 + c_x$, $b_w = (2\sigma(\Delta w))^2 \times a_w$等）、letterbox反变换回原始图像坐标。

### 5.3　C++推理框架设计

板端C++框架由四个核心模块组成，职责划分清晰：

`main_video.cc`为主循环控制模块，包含视频读取、帧处理流水线和输出视频写入。主循环逻辑：读帧 → OpenCV NEON letterbox预处理 → rknn_run NPU推理 → 获取INT8输出 → decode_3branch后处理 → ByteTrack跟踪更新 → 绘制HUD（检测框、类别标签、跟踪ID、FPS） → 写帧。预处理中使用两个预分配缓冲区（src_buf和dst_buf）避免帧间内存分配开销。

`rknn_detector.cc`封装了RKNN API的初始化、输入设置和推理调用。`RknnAppContext`结构体封装了推理所需全部上下文：rknn_context句柄、output_attrs量化参数数组（每个branch的scale和zp，用于CPU端反量化）、output_float_bufs反量化结果缓冲区、input_mem非缓存共享输入内存等。

`postprocess.cc`实现了3-branch解码、NMS和坐标变换。NMS采用标准的按类别分组IoU过滤策略。

**letterbox尺度计算**：$\text{scale} = \min(W_m/W_f, H_m/H_f)$，$\text{pad}_x = (W_m - W_f \times \text{scale})/2$，$\text{pad}_y = (H_m - H_f \times \text{scale})/2$。保持原始宽高比，灰度值114填充（与YOLOv5训练时一致，避免训练-推理不一致）。$W_m=640, H_m=640$为模型输入尺寸，$W_f, H_f$为视频帧原始尺寸。

**板端ByteTrack实现**（ByteTrackAlignTracker类）：

（1）**双阈值对齐设计**——high_threshold设为与conf_threshold相同（0.25），而非ByteTrack原始论文中的0.6-0.7。原因：板端INT8量化后，有效检测的置信度普遍低于PC端FP32值，若high_threshold过高（如0.6），大量有效检测将被降级为低分组，第一轮高分关联的候选池过小，关联效率下降。将high_threshold=conf_threshold确保所有通过置信度阈值的检测均参与第一轮高分关联。

（2）**KF参数严格对齐**——$\sigma_{\text{pos}}=1/20$、$\sigma_{\text{vel}}=1/160$，位置过程噪声和观测噪声相等（$R_{\text{pos}} = Q_{\text{pos}}$），与PC端Python实现完全一致。

（3）**类别投票机制**——轨迹维护class_votes[]计数器，每次检测更新时对观测类别投票+1，最终输出类别取多数投票结果。避免因单帧检测类别抖动（INT8量化可能导致person/car置信度排序偶尔翻转）导致跟踪框类别标签闪烁。

（4）**置信度EMA平滑**——$\text{conf}_t = 0.7 \times \text{det.score} + 0.3 \times \text{conf}_{t-1}$，避免跟踪框的置信度数值在帧间大幅跳动。

（5）**尺寸EMA平滑**——跟踪框的中心坐标使用KF滤波值（响应快、跟随准确），宽高尺寸使用EMA平滑值（$\alpha_{\text{ema}} = 0.3$），避免红外边界模糊导致的帧间尺寸跳变。

### 5.4　NPU性能优化

五项优化措施将板端端到端FPS从基准的25.2逐步提升至33以上，累计提升幅度31%。

**措施1：RKNN_FLAG_ENABLE_SRAM**。在rknn_init时设置SRAM标志位，将模型权重和中间特征图优先缓存到NPU片上SRAM（512KB）。SRAM访问延迟约1-2个时钟周期，远低于LPDDR4的约100个时钟周期。对于YOLOv5s的前几层卷积（通道宽度32-64，特征图尺寸较大），中间结果可部分缓存在SRAM中，减少DDR带宽争用。效果：NPU推理时延从~31ms降至~28ms。

**措施2：非缓存输入内存 + 禁止Cache Flush**。使用rknn_create_mem2分配non-cacheable输入内存区域，并设置RKNN_FLAG_DISABLE_FLUSH_INPUT_MEM_CACHE标志。常规流程中，CPU写入输入缓冲区后需要执行cache flush确保NPU看到最新数据。non-cacheable内存绕过CPU cache，CPU写入直接可见于NPU，省去了每帧推理前的cache同步操作。效果：节省约1ms/帧。

**措施3：logit预滤波**（详见5.2节）。在sigmoid之前利用反函数将阈值映射到logit空间，跳过约98%背景Anchor的完整解码。效果：CPU后处理从约7ms降至约4ms，是所有优化中单项收益最大的措施。

**措施4：rknn_set_core_mask(RKNN_NPU_CORE_ALL)**。绑定NPU任务到所有可用NPU核心，防止OS调度器在核心间迁移NPU任务导致的上下文切换开销。效果：约0.5ms稳定性提升。

**措施5：编译器优化（-O3 -march=armv8-a+simd -ffast-math）**。-O3启用最高级别编译优化（循环展开、内联扩展等）；-march=armv8-a+simd启用ARM NEON SIMD向量化，对letterbox预处理和反量化中的像素级操作加速明显；-ffast-math允许浮点运算的近似优化（如将除法替换为乘以倒数），对sigmoid等函数的计算有轻微加速。效果：约0.5-1ms综合提升。

### 5.5　板端跟踪参数迭代优化

板端NPU的INT8推理存在固有的非确定性——同一帧图像在多次推理中可能产生不同数量的检测框（浮动范围约±5%）。这种非确定性源于NPU内部定点运算的调度顺序差异和中间结果的截断舍入差异。直接将PC端（FP32精度）的跟踪参数移植到板端（INT8精度）并不能保证最优效果，需要在板端实际环境下进行参数迭代优化。

**优化方法论**。评测指标：唯一轨迹ID数（越少表示跟踪越稳定）和轨迹展示总数（所有帧中跟踪框出现的总次数，越高表示可视覆盖率越好）。实验控制：每轮仅调整1-2个参数，其余保持不变，确保可归因。固定检测参数conf=0.25、nms=0.45。测试视频：seq006（221帧）和seq009（565帧）。NPU非确定性处理：同一配置运行多次观察趋势性变化，只有方向一致且幅度超过噪声范围的变化才被视为有效改进。

**基线配置（PC端参数直接移植）**：

```
match_iou = 0.30, second_match_iou = 0.20, min_hits = 3
visible_lag = 1, high_threshold = 0.50, max_age = 30
```

基线测试结果：seq006唯一ID=49，展示=2353；seq009唯一ID=74，展示=5170；FPS=27.6。

**Round 1（宽松IoU匹配阈值）**。调整：match_iou从0.30降至0.25，second_match_iou从0.20降至0.15。调整思路：板端INT8量化导致检测框坐标精度较PC端FP32有所下降，表现为检测框位置在帧间的微小偏移（约1-2像素），这种偏移直接降低了帧间IoU得分。适当放宽IoU阈值容忍这一额外偏移。R1结果：seq006唯一ID=46（-6%），方向正确。

**Round 2（加速确认+延长展示）——最终选用**。在R1基础上调整：min_hits从3降至2，visible_lag从1增至3。调整思路：min_hits=2降低了因INT8帧间漏检导致新轨迹无法确认的概率——PC端FP32下连续3帧匹配合理，但INT8非确定性可能导致中间帧漏检，2帧确认更适应这一特性。visible_lag=3使轨迹丢失后继续显示3帧KF预测位置，平滑单帧漏检造成的跟踪框突然消失。

R2结果：seq006唯一ID=42（**-14%** vs基线49），展示=2766（**+18%** vs基线2353）；seq009唯一ID=66-73（**-1~11%** vs基线74），展示=5465（+6%）；FPS=27.6-27.8，无性能退化。R2在两个序列上均取得了唯一ID下降和展示覆盖率提升的双向改善。

**Round 3（降低高阈值）——已拒绝**。在R2基础上尝试：high_threshold从0.50降至0.45。R3结果：seq006唯一ID=57（+35% vs R2），大幅恶化。原因：high_threshold=0.50是过滤INT8量化噪声的有效边界。0.45-0.50区间内的检测包含大量量化噪声引起的假阳性，这些假阳性检测的边界框位置不稳定，进入高分池后无法与已有轨迹有效匹配，反而创建大量短命新轨迹。结论：high_threshold不应低于0.50。

### 5.6　本章小结

本章完成了从PC端训练模型到RV1126B板端实时推理的完整工程链路，取得以下成果：

## 第7章　综合实验与系统演示

### 7.1　系统功能验证

使用exp07_eiou RKNN模型（普通量化）在RV1126B板端处理两个测试序列，从视频输入到最终输出的完整链路为：

```
视频帧读取(~0.5ms) → NEON letterbox预处理(~1.5ms) → RKNN NPU INT8推理(~26ms) 
→ 3-branch后处理解码(~4ms) → ByteTrack跟踪更新(~0.5ms) 
→ HUD绘制(~0.3ms) → MP4帧写出(~0.5ms) → 总计~33ms/帧 ≈ 30FPS
```

功能验证结果如下：

**检测功能验证**。模型正确检测person和car两类目标，检测框与目标灰度亮区吻合，类别标签准确。在seq009密集行人场景中，同时存在5-8个行人目标的帧中，各个行人均被独立检测并分配独立的检测框，说明NMS去重和多尺度检测正常工作。远距离小行人（约20-30像素高度）的检测率较近距离目标有所下降，这与INT8量化对弱信号目标的精度影响一致。

**跟踪功能验证**。ByteTrack为每个检测到的目标分配稳定的ID号，跟踪框在连续帧中平滑过渡。在seq006中，一辆快速横向运动的车辆保持ID不变地被跟踪超过80帧。在seq009中两个行人短暂交叉遮挡后，各自恢复原有ID，通过KF的速度预测和IoU关联实现了遮挡恢复。R2参数优化后的效果明显优于基线配置——跟踪框消失和重新出现的频率降低（visible_lag=3的缓冲效果），短暂漏检后轨迹能更快确认（min_hits=2的加速效果）。

**性能功能验证**。帧率稳定在27-28 FPS（含ByteTrack跟踪），纯检测模式33+ FPS，均超过25 FPS实时阈值。连续处理565帧seq009序列期间帧率波动不超过±1 FPS，无明显卡顿现象。

**GUI全流程验证**。在可视化管理系统中完成"选择EIoU普通量化模型→SSH远程编译→ADB上传至板端→执行板端推理→SFTP拉取输出视频→VideoPlayer播放"的全流程操作。各环节的日志信息实时显示在LogPanel中，编译和推理耗时以进度条形式呈现。验证了Windows→Ubuntu→RV1126B三层联动能力的完整性和可靠性。

**检测数量差异分析**。板端检测数量较PC端减少18-21%，这一差距可从两个维度理解：

第一，**量化截断效应**。INT8量化将FP32参数的32位精度压缩至8位，量化噪声$\Delta^2/12$使边界检测（置信度在阈值0.25附近的检测框）的一部分跌破阈值被过滤。这些被过滤的检测主要是远距离小目标和低对比度目标——它们在FP32下的置信度本就不高，量化噪声使其从"勉强通过"变为"略低于阈值"。

第二，**对核心目标的影响有限**。近距离、高对比度的目标（如画面中间距离的行人和车辆，置信度通常在0.6-0.9范围内）在量化后置信度仍远高于阈值，几乎不受影响。因此18-21%的检测数量下降主要体现在"边缘"检测的丢失，对跟踪质量的实际影响（唯一ID增加约14%）小于检测数量的下降幅度。

### 7.3　典型场景分析

**seq009行人密集场景（565帧）**。该序列包含城市人行道的密集行人混合运动场景，主要挑战包括：

（1）**频繁遮挡**。行人之间在交叉行走时频繁产生部分遮挡或完全遮挡。ByteTrack的双阈值机制在第二轮关联中发挥关键作用——遮挡导致行人检测置信度从0.7+降至0.3-0.5（部分遮挡）或完全丢失（完全遮挡），低分二次关联能够挽救0.3-0.5的部分遮挡检测，维持轨迹连续性。完全遮挡的情况则依赖KF预测+lost缓冲（max_age=30帧），在行人重新出现后通过reactivate匹配恢复轨迹。

（2）**密集目标区分**。5-8个行人在相邻区域内同时运动时，匈牙利算法的全局最优匹配和IoU阈值控制（match_iou=0.25）在大多数情况下能正确关联相邻帧中的对应目标。但当两个行人长时间并行且间距极小（<10像素）时，IoU关联可能出现误匹配，产生ID Switch。R2配置下seq009唯一ID为66-73，较基线74有所改善但波动范围较大（因NPU非确定性）。

（3）**目标进出画面管理**。行人从画面边缘进入时，前几帧的检测框通常只覆盖目标的一部分（截断效应），ByteTrack为其创建tentative轨迹，min_hits=2帧后确认。行人离开画面时，检测逐渐消失，lost缓冲机制确保轨迹不会立即删除。

**seq006车辆快速运动场景（221帧）**。该序列包含一辆快速横向运动的车辆，帧间位移约20像素（640×512分辨率下约3%画面宽度）。

KF状态向量包含速度分量$[\dot{c}_x, \dot{c}_y]$，对匀速运动的预测有效。不使用KF预测时（仅使用上一帧检测框位置），快速运动车辆的帧间IoU仅约0.6-0.7（因移动20像素导致重叠面积显著减小）。使用KF预测后（基于速度外推下一帧位置），预测框与实际检测框的IoU可达0.85+，关联成功率大幅提升。

## 第8章　总结与展望

### 8.1　主要工作总结

本文围绕红外多目标检测与跟踪系统的关键技术问题，开展了从算法改进到嵌入式部署再到系统集成的全栈研究工作，取得了以下主要成果：

**（1）系统性消融实验与检测模型改进**。设计并执行13组严格控变消融实验，覆盖轻量化主干（Ghost Module、ShuffleNet）、注意力机制（SE/CBAM、CoordAtt）和损失函数优化（SIoU、EIoU）三大类改进策略的独立及组合效果。核心发现：EIoU损失函数在不增加参数量的前提下取得全消融最优精度mAP@0.5=0.817（+0.85pp vs Baseline），其通过独立惩罚宽度和高度误差有效改善了红外图像中边界模糊目标的回归精度；注意力机制在小模型+红外数据条件下无正向增益；组合改进策略存在负迁移效应。

**（2）多目标跟踪算法的系统评估**。完成3种跟踪算法在5种检测模型上的全交叉对比（15组配置）。ByteTrack以ID Switch=26远低于DeepSORT（128）和CenterTrack（87），在红外场景中系统验证了纯运动关联策略优于外观辅助方法这一重要结论。DeepSORT的Re-ID外观特征在红外灰度图中不仅无效，更产生了负面干扰。

**（3）RV1126B嵌入式部署的完整工程链路**。实现了从PyTorch到RKNN的模型量化转换（对比普通量化、KL散度量化，说明MMSE排除原因，分析EIoU模型与普通量化的适配问题），设计了完整的C++推理框架，执行了五项NPU性能优化使FPS从25.2提升至33+（+31%），在板端环境下完成3轮控制变量跟踪参数迭代优化（唯一ID降低14%，展示覆盖率提升18%）。

**（4）全流程可视化管理系统**。基于PyQt5实现五大功能模块，通过SSH/ADB三层联动覆盖从数据处理到板端部署的完整研究流程。

### 8.2　不足与展望

本文的研究存在以下不足，也指明了未来可深入的方向：

**（1）纵向深入优化空间**。本文采取横向对比路线确定了EIoU为最优方向，但未在EIoU基础上进行纵向深入调参。后续可探索：EIoU中宽高惩罚权重的调节对特定类别的精度影响；针对FLIR数据集特性的定制化数据增强策略（如红外噪声模拟、辐射强度随机波动等）；针对person和car两类目标精度不均衡的类别平衡策略。

**（2）量化感知训练（QAT）**。当前采用训练后量化（PTQ），量化精度损失无法在训练阶段补偿。引入QAT使模型在训练过程中模拟INT8量化效应，有望缩小板端精度差距，将检测数量差异从当前的18-21%降低至10%以内。

**（3）红外域Re-ID特征适配**。当前实验证实基于可见光训练的Re-ID在红外域失效。后续可探索基于红外特征的Re-ID方案：利用目标辐射强度分布模式（不同个体的躯干-四肢辐射比例可能不同）或运动模式特征（步态、速度）构建红外域的外观/行为描述子。

**（4）RTSP实时流接入**。当前系统处理离线视频文件，后续可接入RTSP视频流或直接连接红外摄像头，实现真正的端到端实时推理。需要解决网络传输抖动、帧丢弃策略和推理-显示同步等工程问题。

**（5）MOT标准评估指标完善**。当前使用的ID Switch Proxy为代理指标，缺乏逐帧轨迹真值标注。后续可为测试序列手工标注或半自动标注真实轨迹ID，计算标准MOTA、MOTP、IDF1等指标，实现与其他论文的绝对精度比较。

**（6）多场景泛化性增强**。当前实验基于FLIR城市道路数据集的两类目标（person、car），后续可扩充训练数据覆盖夜间、雨雾、工业巡检等多场景，增加bicycle、animal等目标类别，并尝试多分辨率训练策略提升跨场景泛化能力。

### B.1　EIoU损失函数实现（YOLOv5 metrics.py节选）

```python
def bbox_eiou(box1, box2, eps=1e-7):
    """计算EIoU损失，独立惩罚宽度和高度误差"""
    b1_x1, b1_y1, b1_x2, b1_y2 = box1.chunk(4, -1)
    b2_x1, b2_y1, b2_x2, b2_y2 = box2.chunk(4, -1)
    w1, h1 = b1_x2 - b1_x1, b1_y2 - b1_y1
    w2, h2 = b2_x2 - b2_x1, b2_y2 - b2_y1
    
    # IoU计算
    inter = (torch.min(b1_x2, b2_x2) - torch.max(b1_x1, b2_x1)).clamp(0) * \
            (torch.min(b1_y2, b2_y2) - torch.max(b1_y1, b2_y1)).clamp(0)
    union = w1 * h1 + w2 * h2 - inter + eps
    iou = inter / union
    
    # 最小外接矩形
    cw = torch.max(b1_x2, b2_x2) - torch.min(b1_x1, b2_x1)
    ch = torch.max(b1_y2, b2_y2) - torch.min(b1_y1, b2_y1)
    c2 = cw ** 2 + ch ** 2 + eps
    
    # 中心点距离
    rho2 = ((b1_x1 + b1_x2 - b2_x1 - b2_x2) ** 2 +
            (b1_y1 + b1_y2 - b2_y1 - b2_y2) ** 2) / 4
    
    # EIoU: 独立的宽度和高度惩罚
    rho_w2 = (w1 - w2) ** 2 / (cw ** 2 + eps)
    rho_h2 = (h1 - h2) ** 2 / (ch ** 2 + eps)
    
    return iou - rho2 / c2 - rho_w2 - rho_h2
```

### B.2　ByteTrack双阈值关联核心代码（bytetrack_tracker.py节选）

```python
def update(self, detections):
    # 按置信度分为高分和低分两组
    high_dets = [d for d in detections if d.score >= self.high_threshold]
    low_dets = [d for d in detections if self.low_threshold <= d.score < self.high_threshold]
    
    # 第一轮：高分检测 vs 所有活跃轨迹
    cost_matrix = build_iou_cost(self.active_tracks, high_dets)
    matched, unmatched_tracks, unmatched_dets = hungarian_match(
        cost_matrix, thresh=self.match_iou_threshold)
    
    for track_idx, det_idx in matched:
        self.active_tracks[track_idx].update(high_dets[det_idx])
    
    # 第二轮：低分检测 vs 未匹配轨迹
    cost_matrix_2 = build_iou_cost(
        [self.active_tracks[i] for i in unmatched_tracks], low_dets)
    matched_2, still_unmatched, _ = hungarian_match(
        cost_matrix_2, thresh=self.second_match_iou_threshold)
    
    for rel_idx, det_idx in matched_2:
        global_idx = unmatched_tracks[rel_idx]
        self.active_tracks[global_idx].update(low_dets[det_idx])
    
    for idx in still_unmatched:
        self.active_tracks[idx].mark_lost()
    
    for det_idx in unmatched_dets:
        self.create_new_track(high_dets[det_idx])
```
