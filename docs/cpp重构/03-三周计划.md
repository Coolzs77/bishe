# C++ 重构三周计划

## 1. 总体策略

你只有 3 周，所以不能把目标定成“大而全”。

这 3 周应该遵循三个原则：

1. 每周都有可运行成果
2. 每天都围绕当前模块补最少量的 C++ 知识
3. 不做和闭环无关的大扩展

最终目标：

1. 第 1 周做完检测
2. 第 2 周做完跟踪
3. 第 3 周做完 Ubuntu 部署与收尾

## 5. 三周中的学习方法

你不需要先把 C++ 学完。

你只需要坚持这套方法：

1. 今天做什么模块，就补这个模块会用到的语言点
2. 不追求面面俱到
3. 每天结束前必须留下能运行或能编译的结果
4. 每周末必须形成阶段成果

## 6. 三周中要严守的限制

1. 不要中途突然加 GUI
2. 不要中途改成多线程流水线
3. 不要中途切 TensorRT
4. 不要因为看到新框架就换方案
5. 不要在检测闭环没跑通前做跟踪
