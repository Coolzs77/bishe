# 项目里能练到的 C++ 知识地图

## 1. 你这次不会是“学语法”，而是在项目里学工程

你当前最需要的不是把教材按章节看完，而是在项目中掌握最常用、最有工程价值的 C++ 能力。

## 2. 第一类：基础语法到工程语法

你会实际用到：

1. 头文件和源文件分离
2. 声明与定义
3. 命名空间
4. const 正确性
5. 引用传参
6. 类、对象、构造、析构

这些能力会出现在：

1. 基础类型定义
2. 配置读取类
3. 检测器封装
4. 跟踪器封装

## 3. 第二类：STL 容器与算法

这是你这次一定会显著进步的部分。

### 重点容器

1. vector
2. deque
3. string
4. unordered_map
5. pair
6. optional

### 重点算法和写法

1. sort
2. find_if
3. remove_if
4. transform
5. lambda

### 在项目中的典型使用

1. vector 存检测框和轨迹
2. deque 存轨迹历史路径
3. unordered_map 管理轨迹状态
4. sort 给候选框按分数排序
5. remove_if 清除失效轨迹

## 4. 第三类：面向对象设计

你会练到：

1. 抽象接口
2. 继承与多态
3. 组合关系
4. 模块边界设计
5. 单一职责原则

你会逐渐明白：

1. 不是所有东西都该写成一个类
2. 不是所有逻辑都该堆在 main 里
3. 接口清晰比函数多更重要

## 5. 第四类：资源管理与 RAII

这是 C++ 和脚本语言体验差异最大的部分之一。

你会练到：

1. 对象生命周期
2. 栈对象和堆对象的取舍
3. 智能指针的使用时机
4. 文件与视频句柄管理
5. 出错时资源自动释放

会对应到这些场景：

1. ONNX Runtime 会话
2. OpenCV VideoCapture 和 VideoWriter
3. 配置对象与模块初始化

## 6. 第五类：工程构建与依赖管理

你会系统接触：

1. CMake 基础
2. 目标、库、可执行程序
3. include 目录管理
4. 外部库链接
5. Debug 与 Release

这个部分的价值非常高，因为它直接决定你能不能在 Ubuntu 20.04 上落地。

## 7. 第六类：调试与定位问题

你会被迫提升：

1. 编译错误定位
2. 链接错误定位
3. 运行时崩溃定位
4. 数值偏差定位
5. Python 与 C++ 结果对照

这是最能快速拉开工程能力差距的一部分。

## 8. 第七类：算法实现能力

在这个项目里，你不是只会调库，还会真正碰到算法实现。

你会接触：

1. IoU 计算
2. NMS
3. 坐标映射
4. 跟踪匹配
5. 卡尔曼滤波或简化运动预测

这部分会把你的 C++ 从“语法层”拉到“逻辑层”。

## 9. 第八类：Linux 与部署能力

这是你后续最有竞争力的部分之一。

你会练到：

1. Ubuntu 环境构建
2. 动态库与运行路径问题
3. 命令行程序组织
4. 性能测试与日志输出
5. 可移植工程思维

## 10. 你这 3 周最应该优先补的 C++ 点

如果只按项目需要排序，优先级建议如下：

1. class 和 struct
2. const 与引用
3. vector 和 string
4. sort 和 lambda
5. enum class
6. 文件读写
7. RAII 基本思想
8. CMake 基础
9. 异常处理
10. filesystem

## 11. 不必现在深挖的内容

下面这些不是当前 3 周的重点：

1. 模板元编程
2. 复杂并发编程
3. 协程
4. 花哨设计模式
5. 过深的标准库细节

你现在需要的是先把工程做出来，而不是把 C++ 全宇宙一次学完。

## 12. 你会感受到的明显提升

如果你按这套路线推进，最明显的提升会是：

1. 从只会写语句，变成会组织模块
2. 从只会运行脚本，变成会构建工程
3. 从看懂代码，变成能独立实现检测和跟踪流程
4. 从 Windows 本地实验，变成能在 Ubuntu 上部署运行

这几项提升都是真实可见的，而且是可以写进你的项目经历里的。
