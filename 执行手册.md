# 执行手册

## 一、当前仓库范围

当前仓库只保留以下实验主线：

- 数据准备
- 单模型训练
- 检测消融实验
- 检测评估
- 多目标跟踪评估
- 结果绘图与汇总

部署、RKNN、嵌入式相关内容已移除。

## 二、训练前提

开始训练前，请先确认以下条件：

1. 已准备 Python 环境并安装 requirements.txt 中依赖。
2. 仓库内存在 yolov5 目录。
3. 默认预训练权重 yolov5/yolov5s.pt 存在。
4. 已生成 data/processed/flir/dataset.yaml。
5. 如需跟踪评估，已准备 data/videos/thermal_test。

## 三、数据准备

### 1. FLIR 数据整理

```bash
python scripts/data/prepare_flir.py --input data/raw/flir
```

输出重点：

- data/processed/flir/images/train
- data/processed/flir/images/val
- data/processed/flir/labels/train
- data/processed/flir/labels/val
- data/processed/flir/dataset.yaml

### 2. 跟踪视频准备

如果输入是连续图像帧，可整理为热视频：

```bash
python scripts/data/images_to_video.py --input data/raw/flir/video_thermal_test/data --output data/videos/thermal_test
```

## 四、配置文件边界

这一步非常关键，避免继续混用：

### 1. 单模型训练配置

- 文件：[configs/train_config.yaml](configs/train_config.yaml)
- 仅供 [scripts/train/train_yolov5.py](scripts/train/train_yolov5.py) 使用
- 用于单个模型的基准训练、补训、断点续训

### 2. 消融实验训练配置

- 文件：[configs/ablation/train_profile_controlled.yaml](configs/ablation/train_profile_controlled.yaml)
- 文件：[configs/ablation/train_profile_optimal.yaml](configs/ablation/train_profile_optimal.yaml)
- 仅供 [scripts/train/train_ablation.py](scripts/train/train_ablation.py) 使用
- controlled 用于严格控变量
- optimal 用于各实验有限度独立调参

### 3. 不应再混用的规则

1. 不要用 train_config.yaml 去控制 train_ablation.py。
2. 不要把消融训练参数回填到 train_config.yaml。
3. 单模型训练和消融训练应分别维护各自配置口径。

### 4. 检测评估配置

- 文件：[configs/eval_detection.yaml](configs/eval_detection.yaml)
- 仅供 [scripts/evaluate/eval_detection.py](scripts/evaluate/eval_detection.py) 使用
- 负责 detection 评估默认模式、数据集、运行参数、批量评估口径和结果保存设置

### 5. 跟踪评估配置

- 文件：[configs/tracking_config.yaml](configs/tracking_config.yaml)
- 仅供 [scripts/evaluate/eval_tracking.py](scripts/evaluate/eval_tracking.py) 使用
- 负责检测器输入、视频来源、输出开关、tracker 默认参数和可视化开关

### 6. 评估阶段不应再混用的规则

1. 不要把 detection 评估默认参数写回 tracking_config.yaml。
2. 不要用 tracking_config.yaml 去控制 eval_detection.py。
3. eval_detection.py 和 eval_tracking.py 的命令行参数只作为临时覆盖，不再承担默认来源。

### 7. 绘图配置

- 文件：[configs/plot_eval_summary.yaml](configs/plot_eval_summary.yaml)
- 仅供 [scripts/evaluate/plot_eval_summary.py](scripts/evaluate/plot_eval_summary.py) 使用
- 文件：[configs/plot_tracking_comparison.yaml](configs/plot_tracking_comparison.yaml)
- 仅供 [scripts/evaluate/plot_tracking_algorithm_comparison.py](scripts/evaluate/plot_tracking_algorithm_comparison.py) 使用
- 两个绘图脚本的命令行参数只作为临时覆盖

## 五、单模型训练

### 1. 按配置文件训练

```bash
python scripts/train/train_yolov5.py --config configs/train_config.yaml
```

### 2. 临时覆盖部分参数

```bash
python scripts/train/train_yolov5.py --config configs/train_config.yaml --epochs 100 --batch-size 16 --device 0
```

### 3. 断点续训

```bash
python scripts/train/train_yolov5.py --resume outputs/weights/infrared_detection/weights/last.pt
```

说明：train_yolov5.py 现在是纯训练入口，不再承担自动克隆仓库、自动下载权重、训练时长估算等工具化逻辑。

## 六、消融实验训练

### 1. controlled 口径

用于论文主表、严格控制变量：

```bash
python scripts/train/train_ablation.py --profile controlled
```

### 2. optimal 口径

用于追求各方法上限性能：

```bash
python scripts/train/train_ablation.py --profile optimal
```

### 3. 只跑某一个实验

```bash
python scripts/train/train_ablation.py --profile controlled --only exp7
```

### 4. 只列出可运行实验

```bash
python scripts/train/train_ablation.py --stage all --list
```

### 5. 临时覆盖 profile 参数

以下覆盖只对当前命令生效：

```bash
python scripts/train/train_ablation.py --profile controlled --epochs 50 --batch 16 --img 640
```

建议：

1. 正式 controlled 结果尽量不要依赖命令行临时覆盖。
2. 如果一套参数要长期保留，应写回对应 profile 配置文件。

## 七、检测评估

### 1. 单模型评估

```bash
python scripts/evaluate/eval_detection.py --config configs/eval_detection.yaml --weights outputs/ablation_study/ablation_exp07_eiou/weights/best.pt
```

输出结构：

- 默认生成 outputs/detection/detection_eval_时间戳/
- 目录内默认保存 summary.json
- 不再额外生成独立的 val_detection 根目录

### 2. 批量评估消融实验

```bash
python scripts/evaluate/eval_detection.py --config configs/eval_detection.yaml --batch-eval --sort-by map5095
```

输出结构：

- 默认生成 outputs/detection/detection_eval_batch_时间戳/
- 目录内默认保存 summary.json
- 目录内默认保存 summary.csv

### 3. 检测结果绘图

```bash
python scripts/evaluate/plot_eval_summary.py --config configs/plot_eval_summary.yaml --input-dir outputs/detection/detection_eval_batch_xxx
```

绘图输出结构：

- 默认读取批次目录下的 summary.csv
- 默认在同目录生成 journal_barline.png、journal_improvement.png、journal_metric_change.png

## 八、跟踪评估

建议流程：

1. 先根据检测评估筛候选检测模型。
2. 只对候选模型跑跟踪，避免口径膨胀。

### 1. 运行单个跟踪器

```bash
python scripts/evaluate/eval_tracking.py --config configs/tracking_config.yaml --weights outputs/ablation_study/ablation_exp07_eiou/weights/best.pt --tracker deepsort --output outputs/tracking/current --half --no-save-vid --no-save-txt --no-overlay
```

### 2. 临时覆盖配置中的评估参数

以下覆盖只对当前命令生效：

```bash
python scripts/evaluate/eval_detection.py --config configs/eval_detection.yaml --weights outputs/ablation_study/ablation_exp07_eiou/weights/best.pt --batch-size 16 --device 0
```

```bash
python scripts/evaluate/eval_tracking.py --config configs/tracking_config.yaml --weights outputs/ablation_study/ablation_exp07_eiou/weights/best.pt --tracker bytetrack --img-size 512 --no-save-vid
```

### 2. 可选 tracker

- deepsort
- bytetrack
- centertrack

### 3. 跟踪结果绘图

```bash
python scripts/evaluate/plot_tracking_algorithm_comparison.py --config configs/plot_tracking_comparison.yaml
```

## 九、推荐执行顺序

### 论文主线顺序

1. prepare_flir
2. train_ablation --profile controlled
3. eval_detection --batch-eval
4. 筛选检测模型
5. eval_tracking
6. plot_tracking_algorithm_comparison

### 辅助顺序

1. train_yolov5 做单模型补训或对照训练
2. train_ablation --profile optimal 做上限探索

## 十、输出目录说明

### 训练输出

- 单模型训练：outputs/weights/实验名
- 消融训练：outputs/ablation_study/实验目录名

### 评估输出

- 检测评估汇总：outputs/detection/批次目录
- 跟踪评估汇总：outputs/tracking

## 十一、当前注意事项

1. 当前数据配置没有独立 test 集。
2. 检测评估结果默认属于验证集结论。
3. 跟踪测试前先固定检测模型，避免口径漂移。
4. 当前不再使用 compare_trackers.py，也不再维护部署链路。
5. controlled 结果更适合写论文主对比；optimal 结果更适合做性能上限补充分析。
