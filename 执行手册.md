# 1.训练
1、基于上次的未训练完的模型继续训练：
```bash
$ python scripts/train/train_yolov5.py --resume outputs/exp2/weights/last.pt --epochs 200 --cache
```
2、基于上次的训练完的模型微调：
```bash
$ python scripts/train/train_yolov5.py --weights outputs/exp2/weights/last.pt --epochs 200 --hyp data/hyps/hyp.finetune.yaml --cache
```

2、评估跟踪效果：

②批量测试：

```bash
python scripts/evaluate/eval_tracking.py --weights outputs/exp5/weights/best.pt --tracker deepsort --data data/videos/thermal_test/
```


你这个脚本现在一共是 3 类用法：单模型指标评估、单模型测速、批量消融评估。  
脚本位置：eval_detection.py

你上一次 `Exit Code: 1` 很可能就是 PowerShell 里把参数拆成了多行导致的，所以你这次要求“单行”是对的。

**1) metric：单模型指标评估（P/R/mAP）**
```bash
python scripts/evaluate/eval_detection.py --mode metric --weights outputs/ablation_study/ablation_exp1_baseline/weights/best.pt --data data/processed/flir/dataset.yaml --batch-size 64 --img-size 640 --device 0 --workers 16 --project outputs/val_detection --name exp1_metric --exist-ok --save-json
```

**2) speed：单模型推理速度评估（FPS, ms/img）**
```bash
python scripts/evaluate/eval_detection.py --mode speed --weights outputs/ablation_study/ablation_exp1_baseline/weights/best.pt --data data/processed/flir/dataset.yaml --img-size 640 --conf-thres 0.25 --iou-thres 0.6 --device 0 --save-json
```

**3) metric + batch-eval：批量评估 stage1（Exp1~Exp5）**
```bash
python scripts/evaluate/eval_detection.py --mode metric --batch-eval --stage stage1 --ablation-dir outputs/ablation_study --weights-name best.pt --data data/processed/flir/dataset.yaml --batch-size 64 --img-size 640 --device 0 --workers 16 --project outputs/val_ablation --name stage1_eval --sort-by map5095 --save-json --save-csv
```

**4) metric + batch-eval：批量评估 stage2（Exp6~Exp9）**
```bash
python scripts/evaluate/eval_detection.py --mode metric --batch-eval --stage stage2 --ablation-dir outputs/ablation_study --weights-name best.pt --data data/processed/flir/dataset.yaml --batch-size 64 --img-size 640 --device 0 --workers 16 --project outputs/val_ablation --name stage2_eval --sort-by map5095 --save-json --save-csv
```

**5) metric + batch-eval：全部阶段一起评估（stage1+stage2）**
```bash
python scripts/evaluate/eval_detection.py --mode metric --batch-eval --stage all --ablation-dir outputs/ablation_study --weights-name best.pt --data data/processed/flir/dataset.yaml --batch-size 64 --img-size 640 --device 0 --workers 16 --project outputs/val_ablation --name all_eval --sort-by map5095 --save-json --save-csv
```

**6) metric：评估 test 集（不是 val）**
```bash
python scripts/evaluate/eval_detection.py --mode metric --weights outputs/ablation_study/ablation_exp4_eiou/weights/best.pt --data data/processed/flir/dataset.yaml --task test --batch-size 64 --img-size 640 --device 0 --workers 16 --project outputs/val_detection --name exp4_test --exist-ok --save-json
```

**7) metric：自定义 JSON 输出文件路径**
```bash
python scripts/evaluate/eval_detection.py --mode metric --weights outputs/ablation_study/ablation_exp5_focal/weights/best.pt --data data/processed/flir/dataset.yaml --batch-size 64 --img-size 640 --device 0 --workers 16 --project outputs/val_detection --name exp5_metric --exist-ok --save-json --output outputs/results/exp5_metric_custom.json
```

**参数速查（你最常用）**
1. `--mode speed|metric`：测速或指标评估。  
2. `--weights`：单模型模式必填；批量模式可不填。  
3. `--batch-eval`：开启批量消融评估（仅 metric）。  
4. `--stage stage1|stage2|all`：批量评估范围。  
5. `--weights-name best.pt|last.pt`：批量评估取哪种权重。  
6. `--sort-by precision|recall|map50|map5095`：批量结果排序字段。  
7. `--save-json`：保存 JSON。  
8. `--save-csv`：批量模式额外保存 CSV。  
9. `--project --name`：val.py 输出目录和实验名。  
10. `--task val|test`：评估集合。
